# FlowX 桌面端框架设计文档（Electron）

## 一、技术选型与架构概述

### 1.1 技术栈

| 层级 | 技术选型 | 说明 |
|:---|:---|:---|
| 桌面框架 | Electron 28+ | 基于Chromium + Node.js，跨平台支持 |
| 前端框架 | Vue 3 + TypeScript | Composition API，类型安全 |
| UI组件库 | Element Plus / Ant Design Vue | 快速构建后台风格界面 |
| 状态管理 | Pinia | 轻量、模块化 |
| 构建工具 | Vite | 快速冷启动，HMR支持 |
| 本地存储 | electron-store | 加密存储用户配置、账号token |
| IPC通信 | @electron-toolkit/preload | 安全的preload脚本桥接 |
| 日志系统 | electron-log | 多级别日志，自动轮转 |
| 更新机制 | electron-updater | 增量更新，支持静默下载 |
| 多开浏览器 | WebContentsView / BrowserView | 独立会话隔离 |

## 三、进程职责划分

### 3.1 主进程（Main Process）

| 模块 | 职责 |
|:---|:---|
| 窗口管理 | 创建/销毁主窗口、多开浏览器窗口、模态对话框 |
| 应用生命周期 | 处理ready、window-all-closed、activate等事件 |
| IPC主监听 | 接收渲染进程请求，调用服务层，返回结果 |
| 平台API调用 | 由于涉及跨域、OAuth回调、文件上传，适合在主进程执行 |
| 本地加密存储 | 存储access_token、refresh_token、用户配置 |
| 自动更新 | 检查更新、下载、安装 |
| 系统托盘 | 最小化托盘、快捷操作菜单 |
| 全局快捷键 | 例如 `Cmd+Shift+F` 打开发布窗口 |
| 协议注册 | 注册 `flowx://` 自定义协议处理OAuth回调 |

### 3.2 预加载脚本（Preload）

- 通过 `contextBridge` 暴露安全的API给渲染进程
- 例如：`window.electron.account.add(platform, authCode)`
- 限制只能调用白名单内的函数，防止远程代码执行

### 3.3 渲染进程（Renderer）

- 纯Vue前端应用，负责界面交互和状态管理
- 不能直接操作Node.js API或文件系统，只能调用preload暴露的API
- 通过IPC与主进程通信（异步）

## 四、安全设计（重点）

### 4.1 沙盒与权限隔离

```typescript
// main/windows/MainWindow.ts
const win = new BrowserWindow({
  webPreferences: {
    preload: path.join(__dirname, '../preload/index.js'),
    sandbox: true,               // 启用沙盒
    contextIsolation: true,      // 上下文隔离
    nodeIntegration: false,      // 禁止渲染进程直接使用Node
    nodeIntegrationInWorker: false
  }
});
```

### 4.2 暴露API白名单

```typescript
// preload/index.ts
import { contextBridge, ipcRenderer } from 'electron';

contextBridge.exposeInMainWorld('electron', {
  account: {
    add: (platform: string, authCode: string) => ipcRenderer.invoke('account:add', platform, authCode),
    list: () => ipcRenderer.invoke('account:list'),
    delete: (id: string) => ipcRenderer.invoke('account:delete', id),
    refreshToken: (id: string) => ipcRenderer.invoke('account:refresh', id)
  },
  publish: {
    submit: (data: PublishRequest) => ipcRenderer.invoke('publish:submit', data),
    getProgress: (taskId: string) => ipcRenderer.invoke('publish:progress', taskId)
  },
  // ... 其他API
});
```

### 4.3 CSP（内容安全策略）

在渲染进程HTML中设置严格的CSP：

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self' https://api.flowx.com">
```

### 4.4 敏感数据存储

- 使用 `safeStorage` 模块加密存储token（Electron 28+）
- 密钥由操作系统管理（Keychain / Credential Manager）

```typescript
import { safeStorage } from 'electron';
const encrypted = safeStorage.encryptString(plainToken);
// 存储到 electron-store
```

### 4.5 多开浏览器会话隔离

每个平台账号独立 `session`，使用 `partition` 隔离：

```typescript
const view = new WebContentsView({
  webPreferences: {
    partition: `persist:account_${accountId}`, // 独立持久化分区
    contextIsolation: true,
    nodeIntegration: false
  }
});
```

## 五、IPC通信协议设计

### 5.1 通信模式

- 渲染进程 → 主进程：`ipcRenderer.invoke(channel, ...args)`（异步请求-响应）
- 主进程 → 渲染进程：`webContents.send(channel, data)`（主动推送，如发布进度）

### 5.2 IPC通道清单

| 通道名 | 方向 | 参数 | 返回 | 说明 |
|:---|:---|:---|:---|:---|
| `account:add` | invoke | platform, authCode | AccountInfo | 添加平台账号 |
| `account:list` | invoke | 无 | AccountInfo[] | 获取所有账号 |
| `account:delete` | invoke | accountId | boolean | 删除账号 |
| `account:refresh` | invoke | accountId | boolean | 手动刷新token |
| `publish:submit` | invoke | PublishRequest | taskId | 提交发布任务 |
| `publish:progress` | invoke | taskId | ProgressInfo | 查询进度 |
| `publish:cancel` | invoke | taskId | boolean | 取消任务 |
| `data:fetchStats` | invoke | accountId, dateRange | StatsData | 获取账号数据 |
| `system:openExternal` | invoke | url | void | 用默认浏览器打开链接 |
| `system:showSaveDialog` | invoke | options | filePath | 保存文件对话框 |
| `system:getAppVersion` | invoke | 无 | string | 获取应用版本 |
| `update:check` | invoke | 无 | UpdateInfo | 检查更新 |
| `update:download` | invoke | 无 | void | 开始下载更新 |

**主动推送通道**：

| 通道名 | 发送时机 | 数据结构 |
|:---|:---|:---|
| `publish:statusChanged` | 发布任务状态变化 | { taskId, status, message } |
| `account:tokenExpiring` | token即将过期（提前24h） | { accountId, platform } |
| `update:downloadProgress` | 更新下载进度 | { percent, bytesPerSecond } |

### 5.3 数据序列化

- 所有IPC传输数据必须可序列化（JSON）
- 不支持函数、Symbol、循环引用
- 使用 TypeScript 接口定义强类型

## 六、窗口管理器设计

### 6.1 主窗口（MainWindow）

- 单例模式，应用启动时创建，关闭时隐藏而非退出（可配置）
- 默认尺寸：1280x800，最小尺寸：1024x768
- 支持深色/浅色主题跟随系统

```typescript
class MainWindow {
  private static instance: BrowserWindow | null = null;
  
  static getInstance(): BrowserWindow {
    if (!MainWindow.instance) {
      MainWindow.instance = new BrowserWindow({ /* 配置 */ });
    }
    return MainWindow.instance;
  }
  
  static show(): void { this.getInstance().show(); }
  static hide(): void { this.getInstance().hide(); }
}
```

### 6.2 多开浏览器窗口（BrowserView）

用于在账号管理面板中嵌入各平台的网页版（如抖音创作服务平台），实现：
- 独立登录态（每账号独立session）
- 可切换账号查看不同平台后台
- 支持自动填写发布表单（高级功能）

实现方式：使用 `WebContentsView` 添加到主窗口的指定区域。

```typescript
class AccountBrowserView {
  private view: WebContentsView;
  private accountId: string;
  
  constructor(accountId: string, url: string) {
    this.view = new WebContentsView({
      webPreferences: {
        partition: `persist:account_${accountId}`,
        preload: path.join(__dirname, '../preload/browserPreload.js')
      }
    });
    this.view.webContents.loadURL(url);
  }
  
  attachToWindow(window: BrowserWindow, bounds: Rectangle) {
    window.contentView.addChildView(this.view);
    this.view.setBounds(bounds);
  }
}
```

### 6.3 模态对话框（Dialog）

- 发布确认框、设置面板、账号授权弹窗
- 使用 `BrowserWindow` 作为模态子窗口，设置 `parent` 和 `modal: true`

## 七、自动更新机制

### 7.1 方案：electron-updater

- 更新服务器：私有OSS/静态文件服务器，提供 `latest.yml` 和安装包
- 支持Windows（NSIS）、macOS（dmg/zip）

### 7.2 更新流程

```
启动 → 检查更新（静默） → 有更新 → 弹窗提示 → 用户确认 → 下载（后台） → 安装提示 → 重启应用
```

### 7.3 实现要点

```typescript
import { autoUpdater } from 'electron-updater';

autoUpdater.setFeedURL({
  provider: 'generic',
  url: 'https://update.flowx.com/win32/x64/'
});

autoUpdater.on('update-available', (info) => {
  mainWindow.webContents.send('update:available', info);
});

autoUpdater.on('update-downloaded', () => {
  autoUpdater.quitAndInstall();
});
```

### 7.4 强制更新场景

对于破坏性API变更，可配置 `forceUpdate: true`，用户必须更新才能继续使用。

## 八、日志与崩溃报告

### 8.1 日志系统（electron-log）

- 自动按日期轮转，保留最近7天
- 日志路径：`%APPDATA%/FlowX/logs/`（Windows）或 `~/Library/Logs/FlowX/`（macOS）
- 分级：error, warn, info, debug

```typescript
import log from 'electron-log';
log.transports.file.level = 'info';
log.info('App started');
```

### 8.2 崩溃报告（Sentry / Electron Crash Reporter）

- 集成 `@sentry/electron`
- 捕获未捕获的异常、Promise rejection、渲染进程崩溃
- 上报时附带用户环境信息（OS版本、Electron版本、是否登录等）

```typescript
import * as Sentry from '@sentry/electron';

Sentry.init({
  dsn: 'https://xxx@sentry.flowx.com/1',
  environment: process.env.NODE_ENV
});
```

## 九、性能优化策略

### 9.1 启动速度

- 使用 `vite` 预构建依赖，减少启动时编译
- 延迟加载非关键窗口（如设置窗口首次打开再创建）
- 使用 `ready-to-show` 事件显示窗口，避免白屏

### 9.2 内存管理

- 多开浏览器窗口超过5个时，自动卸载不可见的WebContents（`webContents.loadURL('about:blank')`）
- 定期清理不再使用的session partition

### 9.3 发布任务并发控制

- 主进程限制同时进行的发布任务数量（如5个并发）
- 超过的任务进入队列，依次执行
