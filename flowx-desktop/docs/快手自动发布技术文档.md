## 一、目标页面结构分析

### 1.1 发布页 URL

```
发布视频: https://cp.kuaishou.com/article/publish/video
发布图文: https://cp.kuaishou.com/article/publish/picture （如需）
账号首页: https://cp.kuaishou.com/
```

### 1.2 页面核心 DOM 结构（快手创作中心特有）

快手使用 **Element UI (Vue.js)** 组件库，DOM 结构有以下特点：

```html
<!-- 上传区域: 页面顶部大按钮
<div class="_upload-btn" data-v-xxxxx>
  <button>上传视频</button>
</div>
<!-- file input 在点击后动态创建 -->
<input type="file" accept="video/*" style="display:none">

<!-- 标题输入框: contenteditable div（不是普通 input）
<div contenteditable="true" class="_input" data-v-xxxxx placeholder="作品标题"></div>

<!-- 正文/描述输入框: contenteditable div -->
<div contenteditable="true" class="_textarea" data-v-xxxxx></div>

<!-- 发布按钮: div + button 组合 -->
<div class="_button_3a3lq_1 _button-primary_3a3lq_60" role="button">
  <span>发布</span>
</div>

<!-- 账号信息（在头像 hover 时出现的 popover，以下为典型结构）
<div class="el-popover el-popper user-info-popper">
  <div class="header-info-card">
    <img class="user-image" src="...">
    <div class="user-name">月初七</div>
    <div class="user-kwai-id"></div>
    <div class="user-cnt">
      <div class="user-cnt__item"> 2<span>粉丝</span></div>
      <div class="user-cnt__item">0<span>关注</span></div>
      <div class="user-cnt__item">9<span>获赞</span></div>
    </div>
  </div>
</div>
```

### 1.3 关键发现

1. **title 是 contenteditable** — 快手的标题输入框不是 `<input>`，而是 `<div contenteditable="true">`。必须用 `innerText` 而非 `value`
2. **上传按钮带 `_upload-btn` class** — 直接 querySelector 即可命中
3. **发布按钮是一个 div 而非 button 标签** — class 格式 `_button_3a3lq_1 _button-primary_3a3lq_60`（有 hash 后缀，需用 `*=` 宽松匹配）
4. **统计数据在 popover 中** — 不 hover 头像就不会渲染出 `user-cnt__item`，但账号首页本身也会显示统计信息（见下文）

## 二、完整发布流程

### 2.1 工厂方法 + 适配器模式调用链

快手平台使用新的 PlatformDispatcher 架构：

```
PublishEngine.submit(req)
  └─> PlatformDispatcher.createExecutor('kuaishou', 'video')
        └─> 调用 kuaishou.ts 的 publishVideo()
              └─> runStandardPublish(win, KUAISHOU_VIDEO_CONFIG)
                    └─> shared.ts 中标准化的 9 步流程
```

所有平台共享同一套 `runStandardPublish`，平台差异完全收敛在 config 中。

### 2.2 标准发布流程（9 步）

```
1. 导航到 https://cp.kuaishou.com/article/publish/video
2. 等待页面稳定（NavigationTracker.waitForStable，~1.5s 静默期）
3. 检测登录状态：检查是否出现创作中心界面（loginKeywords 匹配）
4. 文件上传（CDP setFileInputFiles）
5. 等待上传/转码完成（poll 轮询 contenteditable 是否出现）
6. 填写标题（native setter + contenteditable innerText）
7. 填写正文/描述 + 话题
8. 点击发布按钮（Shadow DOM 穿透 + 坐标点击）
9. 点击后验证：检测 URL 跳转到 /article/manage/video?status=2&from=publish
```

### 2.3 关键配置（KUAISHOU_VIDEO_CONFIG）

```typescript
{
  platform: 'kuaishou',
  meta: {
    publishUrl: 'https://cp.kuaishou.com/article/publish/video',
    homeUrl: 'https://cp.kuaishou.com/',
  },
  detectLoggedIn: (win) => evalJS(win, `
    document.body.innerText.includes('创作中心') ||
    document.body.innerText.includes('发布作品') ||
    document.body.innerText.includes('数据中心')
  `, 'detect-login', log),
  publishKeywords: ['发布作品', '立即发布', '发布', '确认发布'],
  enableConfirmStep: false,
  enablePostClickVerify: true,
  fillWaitMs: 1500,
}
```

## 三、核心难点与解决方案

### 难点 1: 标题/正文是 contenteditable 而非 input

**问题**: 快手的标题和描述输入框是 `<div contenteditable="true">`，而不是标准的 `<input>` 或 `<textarea>`。直接设置 `value` 属性不生效。

**解决方案**: 在 `buildFillTitle` / `buildFillContent` 中针对 contenteditable 元素使用 `innerText` 赋值，并触发 `input` 事件让 React/Vue 感知：

```javascript
// 对 contenteditable 元素:
el.innerText = title;
el.dispatchEvent(new Event('input', { bubbles: true }));
el.dispatchEvent(new Event('change', { bubbles: true }));
el.dispatchEvent(new Event('blur', { bubbles: true }));
```

在 shared.ts 中实现的策略优先级:
1. **contenteditable 元素**（最高优先级，+80 分基础分）
2. 有 `placeholder` 或 `title` 属性的 input
3. 通用 input/textarea 兜底

### 难点 2: 发布按钮是 div 而非 button

**问题**: 快手的发布按钮:

```html
<div class="_button_3a3lq_1 _button-primary_3a3lq_60" role="button">
  <span>发布</span>
</div>
```

**解决策略链**:
1. `buildPublishButtonClicker` 策略 1: 精确匹配 `[class*="_button-primary_"]` — 命中主按钮 div
2. 策略 2: 精确匹配 `[class*="_button_"]` — 命中所有 _button_* 元素
3. 策略 3: 关键词文本 "发布作品"/"立即发布"/"发布"
4. 兜底: Shadow DOM 穿透 + 坐标点击（CDP 协议）

### 难点 3: 粉丝/关注/获赞数提取 - Element UI Popover

**问题**: 快手的统计信息有两个位置:

**位置 A — 账号首页 header-info-card**:
```html
<div class="header-info-card">
  <div class="user-cnt">
    <div class="user-cnt__item"> 2<span>粉丝</span></div>
    <div class="user-cnt__item">0<span>关注</span></div>
    <div class="user-cnt__item">9<span>获赞</span></div>
  </div>
</div>
```

**位置 B — hover 头像时弹出的 Popover**:
```html
<div class="el-popover el-popper user-info-popper" style="...display:none...">
  <!-- 与位置 A 相同的结构 -->
</div>
```

**解决方案（完全同步的 JS 注入）**:

```javascript
// 三层策略:
// 1. 直接查询 [class*="user-cnt__item"] — 最可靠
// 2. 在 .header-info-card / [class*="user-cnt"] / .el-popover / .user-info-popper
//    容器内扫描所有 div/span/p
// 3. 兜底: 全页所有 div/span 文本扫描
// 解析逻辑: 每个元素 textContent.trim() 中找
//   - 关键词: "粉丝" / "关注" / "获赞" / "点赞"
//   - 数字格式: 支持 123 / 1.2万 / 1,234 / 12k
// 示例: "2粉丝" -> fansCount = 2
// 示例: "1.2万关注" -> followCount = 12000
```

**实现要点**:
- **完全同步**: 不使用 Promise/setTimeout 等异步结构，避免 `Script failed to execute` 错误
- **多层容错**: element UI class 名有 hash（如 `_button_3a3lq_1`），使用 `class*=` 宽松匹配
- **数字解析**: `parseCount()` 函数处理 "1.2万" -> `12000` 的格式
- **调试字段**: `_debug { samples, hit, errors, cookieUserId }` — 实际提取到的文本样本便于排错

### 难点 4: 快手号（platformAccountId）的获取

**问题**: 页面上的 `user-kwai-id` 元素通常为空（快手不直接显示用户ID）。

**解决方案**: 从 cookie 解析:

```javascript
// 在 cookies 中查找以下字段（按优先级排序）:
// 1. user_id
// 2. kuaishou_id
// 3. kwai_id
// 4. uid / uid_key
// 要求: 值必须匹配 /^[A-Za-z0-9_\-]+$/（纯ID，不含特殊字符）
```

### 难点 5: 上传后状态判断 - 需要区分 "转码中" vs "已就绪"

**问题**: 上传视频后，快手需要 30-60 秒转码，期间需要等待。

**解决方案**: `waitForUploadComplete` 中的轮询策略:

```javascript
// 轮询频率: 每次间隔 4s
// 判定为 ready 的条件（满足任一即可）:
//   1. 页面出现 contenteditable 元素 → 进入编辑阶段
//   2. 页面有 title/作品描述 等关键词
//   3. 页面有 标题 input
// 判定为 uploading 的条件:
//   1. body 含 "上传中/正在上传/处理中/转码中/解析中"
//   2. 还没有编辑元素
```

## 四、发布后验证逻辑

### 4.1 成功跳转检测

点击发布按钮后，成功的 URL 跳转格式:

```
https://cp.kuaishou.com/article/manage/video?status=2&from=publish
                        ^^^^^^^^^^^^^^^  ^^^^^^^^  ^^^^^
                        管理页            已发布     从发布页来
```

在 `runStandardPublish` 中的验证逻辑:
1. 点击后等待 `NavigationTracker.waitForStable(800, 15000)`
2. 检测当前 URL 是否包含 `/article/manage/video`
3. 若 URL 未变化，启用 enablePostClickVerify=true 再做一次内容验证

### 4.2 二次点击处理

快手有时会弹出"确认发布"的二次确认:

```
策略:
  1. 首次点击"发布"
  2. 检测弹窗文本 "确认发布" 或 "确定"
  3. 再次点击确认按钮
  4. 等待 URL 跳转到 manage 页面
```

## 六、排错指南

### 6.1 常见失败模式

| 失败现象 | 可能原因 | 排查日志 |
|----------|----------|----------|
| extractPageInfo 返回 `{fansCount:null,...}` | 页面尚未加载统计数据；或 class 名变化 | 检查 `_debug.samples` — 实际命中的文本是什么 |
| 标题没被填入 | contenteditable 元素定位失败 | `[fill-title]` 返回的 `tag/method/totalCandidates` |
| 上传完成后一直是 uploading=true | 轮询条件可能太严格 | `[poll]` 轮询中的 `status/message` |
| 点击发布后 URL 没变 | 点击的元素不对 | `[click-*]` 中的 `clicked:true/false`，检查 `tag/classStr` |
| verify-* 报错 `Script failed to execute` | evalJS 执行报错（异步 JS 问题） | 检查 verify 脚本是否有 Promise/setTimeout 结构 |

### 6.2 关键日志关键字

```
[kuaishou] extractPageInfo: 关注返回的 _debug.samples 字段
[kuaishou][eval] [upload]: 上传按钮点击结果
[kuaishou][eval] [poll-content-fields #n]: 上传轮询
[kuaishou][eval] [fill-title]: 标题填写
[kuaishou][eval] [click-发布作品]: 发布按钮点击
[kuaishou][submit] ✅ URL 已变化: 发布成功
```

### 6.3 如何确认点击到的是什么元素

在 `buildPublishButtonClicker` 返回值中包含以下字段:

```
- tag: 'div' / 'button' / 'a'         # 元素标签
- classStr: '_button_3a3lq_1 ...'       # 完整 class
- text: '发布'                            # 元素文本内容
- score: 5000                           # 评分（class 匹配最高）
- source: 'class:[class*="_button-primary_"]'  # 命中的策略
```

## 七、平台特有注意事项

### 7.1 快手 Anti-Bot 特征

1. **User-Agent 要真实**: 发布窗口使用标准 Chrome/Macintosh UA，不要用 Electron 默认 UA
2. **cookie 注入要在首次导航前完成**: 否则会被要求重新登录
3. **不加速上传/审核流程**: 遵守平台自然节奏

### 7.2 类名 hash 后缀问题

快手的 class 命名格式为 `_button_3a3lq_1 _button-primary_3a3lq_60`，末尾 `3a3lq_60` 是 hash，会随版本变化。

**解决**: 使用 `[class*="_button-primary_"]`（前缀匹配）而非完整类名。

### 7.3 display:none 的 Popover 问题

Element UI popover 默认 `display:none`。hover 才渲染内容。如果 账号首页没有直接显示 user-cnt（可能被收起或在其他位置），导致 extractPageInfo 返回 null 时:

- 优先查询 page 中 `class*="user-cnt__item"` 的元素（账号首页的可见位置）
- 容器内扫描 `.header-info-card`（hover 时弹出的卡片可能存在但被隐藏）
- 全页兜底扫描（确保不遗漏任何包含粉丝/关注/获赞关键词的位置）

## 八、扩展: 快手图文发布

### 8.1 入口

```typescript
// kuaishou.ts publishImage():
//   - 导航到 https://cp.kuaishou.com/article/publish/picture 或
//     https://cp.kuaishou.com/article/publish/image（以实际为准）
```

### 8.2 与视频发布的区别

| 区别 | 视频 | 图文 |
|------|------|------|
| URL | `/article/publish/video` | `/article/publish/picture`（或 image）|
| 上传文件类型 | video/* | image/* |

| 封面选择 | 自动生成可选 | 第一张图片通常作为封面 |
