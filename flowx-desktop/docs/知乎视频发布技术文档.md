## 三、完整发布流程

知乎视频发布共 **10 步**，全程使用 `BrowserWindow` + CDP 实现：

```
步骤 1: 创建独立窗口（账号隔离 partition）
步骤 2: 加载 upload-video 页面
步骤 3: 导航跟踪器等待页面稳定
步骤 4: 登录态检测（cookie + 页面元素双重验证）
步骤 5: CDP 注入视频文件到 input.VideoUploadButton-fileInput
步骤 6: 等待上传完成（表单 + 封面 + 标题三重探针）
步骤 7: 填写标题（textarea，触发 React input 事件）
步骤 7.5: 选择视频标记【必填】（打开 Modal → 等待渲染 → 选选项 → 确认）
步骤 8: 填写描述（Draft.js contenteditable + execCommand）
步骤 9: 点击发布按钮（.VideoUploadForm-submitButton）
步骤 10: 等待发布结果（URL 变化 + 成功文本轮询）
```

### 3.1 步骤详解

#### 步骤 1-3：窗口创建与页面加载

- 使用 `persist:account_{id}` 作为 `partition`，实现账号间完全隔离
- 通过 `NavigationTracker` 等待页面静默（1.5s 无新请求），确保 DOM 稳定
- 导航跟踪器详情见设计文档第五章

#### 步骤 4：登录态检测

优先级：
1. **Cookie 检测**：检查 `z_c0` cookie 是否存在（知乎登录核心标识）
2. **页面元素检测**：查找 `.AppHeader-profile` 或 `.GlobalSideBar` 等登录后特有元素
3. **未登录处理**：弹窗提示用户手动扫码登录，超时 120 秒

#### 步骤 5：CDP 文件上传

```typescript
// 通过 CDP 设置文件输入框的文件
const result = await client.send('DOM.getDocument', { pierce: true });
// 定位 input.VideoUploadButton-fileInput
// 调用 DOM.setFileInputFiles 注入文件
```

知乎的文件输入框是普通的 `<input type="file">`，直接使用 CDP `DOM.setFileInputFiles` 即可，无需特殊处理。

#### 步骤 6：上传完成检测

使用 **三重验证探针**（每 3 秒轮询，最长 5 分钟）：

```javascript
// 探针脚本检测以下三个条件同时满足：
// 1. 表单容器存在：document.querySelector('.VideoUploadForm')
// 2. 封面图存在且有 src：img.VideoUploadForm-image
// 3. 标题输入框存在且有值：textarea[name="title"]
```

只有三个条件全部满足，才认为上传完成（包括转码、封面生成等后台处理）。

#### 步骤 7：标题填写

```typescript
function buildFillTitleScript(title: string): string {
  return `
    (function() {
      var ta = document.querySelector('textarea[name="title"]');
      if (!ta) return { filled: false, error: 'no-title-textarea' };
      
      // 截断到 50 字符
      var maxLen = 50;
      var text = ${JSON.stringify(title)}.slice(0, maxLen);
      
      // 设置值 + 触发 React 受控组件更新
      var nativeSetter = Object.getOwnPropertyDescriptor(
        window.HTMLTextAreaElement.prototype, 'value'
      ).set;
      nativeSetter.call(ta, text);
      ta.dispatchEvent(new Event('input', { bubbles: true }));
      ta.dispatchEvent(new Event('change', { bubbles: true }));
      
      return { filled: true, length: text.length };
    })();
  `;
}
```

**关键点**：知乎标题是 React 受控组件，必须使用 `nativeSetter` + `input/change` 事件才能触发 state 更新，否则 React 重渲染时会被清空。

#### 步骤 7.5：视频标记选择（必填）

这是最复杂的一步，采用**三步异步流程**：

```
点击触发按钮 → 等待 Modal 渲染 → 选择选项 + 点击确认
```

**为什么不能一步完成？**
- 知乎的 Modal 是点击后才动态创建并挂载到 DOM 的
- 同步脚本中点击后立即查询，DOM 中还没有 Modal 元素
- 必须在外层控制：点击 → 等待 → 再查询

**具体实现：**

```typescript
// 外层循环（最多重试 3 次）
for (let attempt = 0; attempt < 3; attempt++) {
  // 第一步：点击打开 Modal
  const openRes = await evalJS(win, buildOpenVideoTagModalScript(), ...);
  if (!openRes.clicked) continue;

  // 第二步：轮询等待 Modal 出现（最多 3 秒，每 300ms 检查）
  let modalReady = false;
  for (let waitStep = 0; waitStep < 10; waitStep++) {
    await sleep(300);
    const check = await win.webContents.executeJavaScript(`...检查 Modal 是否可见...`);
    if (check.exists && check.visible) { modalReady = true; break; }
  }
  if (!modalReady) continue;

  // 第三步：在 Modal 中选择选项并确认
  const selectRes = await evalJS(win, buildSelectAndConfirmVideoTagScript(preferredTag), ...);
  if (selectRes.confirmed && selectRes.selected) {
    // 成功
    break;
  }

  // 失败：关闭 Modal 后重试
  await closeModalIfOpen(win);
  await sleep(500);
}
```

**触发 Modal 的三种方式（按优先级）：**
1. `.VideoUploadForm-videoTypeSelectOverlay[aria-label="选择视频标记"]` — 覆盖层按钮
2. `.VideoUploadForm-select [role="combobox"]` — 选择框内的 combobox 按钮
3. 文本匹配：包含"选择标记"或"视频标记"的可点击元素

**选项选择策略（按优先级）：**
1. 如果已有选中的选项，保持不变
2. 如果传入了 `preferredTag`（来自 `request.category` 或第一个 `tag`），尝试匹配
3. 否则选择第一个非"内容无需标注"的选项（更符合平台推荐）

**确认按钮查找：**
- 在 `.ModalButtonGroup` 中查找文本含"确认"且不含"取消"的按钮
- 避免误点"取消"按钮

#### 步骤 8：描述填写（Draft.js 富文本）

知乎的描述输入框使用 **Draft.js** 富文本编辑器，结构为：

```html
<div class="public-DraftEditor-content" contenteditable="true" role="textbox">
  <div data-contents="true">
    <div class="Editable-unstyled" data-block="true">
      <div class="public-DraftStyleDefault-block">
        <span><br data-text="true"></span>
      </div>
    </div>
  </div>
</div>
```

**填写方案：** 使用 `document.execCommand('insertText')` 模拟原生输入

```typescript
function buildFillContentScript(content: string): string {
  return `
    (function() {
      var editor = document.querySelector('.public-DraftEditor-content');
      if (!editor) return { filled: false, error: 'no-editor' };
      
      // 截断到 1000 字符
      var maxLen = 1000;
      var text = ${JSON.stringify(content)}.slice(0, maxLen);
      
      // 聚焦编辑器
      editor.focus();
      
      // 清空现有内容
      document.execCommand('selectAll', false, null);
      document.execCommand('delete', false, null);
      
      // 插入文本
      document.execCommand('insertText', false, text);
      
      // 触发必要事件
      editor.dispatchEvent(new Event('input', { bubbles: true }));
      editor.dispatchEvent(new KeyboardEvent('keyup', { bubbles: true }));
      
      return { filled: true, length: text.length };
    })();
  `;
}
```

**为什么用 execCommand 而不是直接设置 innerHTML？**
- Draft.js 有自己的内部数据模型（EditorState）
- 直接修改 DOM 不会更新内部状态，React 重渲染时会丢失
- `execCommand('insertText')` 会触发完整的编辑事件管线，Draft.js 能正确捕获并更新 state

#### 步骤 9：点击发布按钮

发布按钮选择器：`button.VideoUploadForm-submitButton`

采用**多选择器评分机制**（与其他平台一致）：
1. 精确匹配：`.VideoUploadForm-submitButton`
2. 类名匹配：包含 `submit` 的按钮
3. 文本匹配：包含"发布视频"或"发布"的按钮
4. 位置匹配：位于表单底部操作区的主要按钮

#### 步骤 10：发布结果检测

发布后进行轮询检测（最长 180 秒），成功判断条件：
1. **URL 变化**：页面跳转到发布成功页或管理页
2. **成功文本**：页面出现"发布成功"、"已发布"、"提交成功"等文本
3. **表单消失**：`.VideoUploadForm` 不再存在

## 四、核心技术难点

### 4.1 视频标记 Modal 的动态渲染问题

**问题**：点击视频标记区域后，Modal 才动态渲染到 DOM 中，同步脚本无法一步完成"点击 → 选择 → 确认"。

**解决方案**：拆分为三步异步流程，外层控制时序
- 第一步：点击触发按钮（JS 脚本）
- 第二步：轮询等待 Modal 出现（外层 sleep + 检查）
- 第三步：选择选项并确认（JS 脚本）

**相关代码**：
- `buildOpenVideoTagModalScript()` — 打开 Modal
- `buildSelectAndConfirmVideoTagScript()` — 选择并确认
- 外层循环重试 3 次，每次失败自动关闭 Modal 后重试

### 4.2 Draft.js 富文本编辑器填写

**问题**：知乎描述使用 Draft.js 框架，直接修改 DOM 不生效。

**解决方案**：使用 `document.execCommand('insertText')` 模拟原生输入，触发完整的编辑事件管线。

### 4.3 React 受控组件标题填写

**问题**：标题 textarea 是 React 受控组件，直接设置 `value` 属性不生效，React 重渲染时会被清空。

**解决方案**：使用 `Object.getOwnPropertyDescriptor` 获取原生 setter，调用后配合 `input/change` 事件触发 React state 更新。

### 4.4 上传完成判定

**问题**：视频上传后需要经历上传、转码、封面生成等多个后台步骤，不能只看 input 的 files 属性。

**解决方案**：三重验证探针
- 表单容器 `.VideoUploadForm` 存在
- 封面图 `img.VideoUploadForm-image` 有 src
- 标题输入框有自动填充的文件名

只有三者同时满足才认为上传完成。

```typescript
const request: PublishRequest = {
```

## 七、排错指南

### 7.1 常见问题

| 现象 | 可能原因 | 排查方法 |
|:---|:---|:---|
| 上传后一直等待 | 视频转码慢或上传失败 | 检查页面是否有错误提示，查看上传探针日志 |
| 标题填了又消失 | React 受控组件事件未触发 | 确认是否使用了 nativeSetter + input 事件 |
| 描述填了不生效 | Draft.js 未捕获输入 | 确认是否使用了 execCommand('insertText') |
| 视频标记 Modal 打不开 | 触发元素选择器失效 | 检查 .VideoUploadForm-videoTypeSelectOverlay 是否存在 |
| Modal 打开但选不中 | 选项选择器失效 | 检查 .VideoUploadForm-videoTypeModalOption 结构 |
| 点击确认没反应 | 确认按钮定位错误 | 检查 ModalButtonGroup 中按钮的文本和类名 |

### 7.2 调试技巧

1. **打开 DevTools**：在发布窗口按 F12 或在代码中设置 `win.webContents.openDevTools()`
2. **查看日志**：控制台会输出每一步的详细日志，包括 evalJS 的返回结果
3. **测试模式**：先用 `testMode: true` 验证表单填写，再测真实发布
4. **手动触发 Modal**：在 DevTools Console 中执行：
   ```javascript
   document.querySelector('.VideoUploadForm-videoTypeSelectOverlay').click()
   ```
   查看 Modal 的 DOM 结构是否与文档一致
