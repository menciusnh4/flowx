# 抖音文章（长文）发布技术文档

> 发布入口：`https://creator.douyin.com/creator-micro/content/upload?default-tab=5`

## 一、文章发布 vs 图文发布 vs 视频发布

| 维度 | 视频发布 | 图文发布 | 文章发布 |
|------|---------|---------|---------|
| 入口 URL | 默认上传页 | `?default-tab=3` | `?default-tab=5` |
| 标题限制 | 55 字 | 80 字 | **30 字** |
| 正文限制 | —（标题即可发布） | 1000 字 | **8000 字** |
| 素材文件 | 视频（必填） | 图片（必填） | **封面图（必填）** |
| 话题标签 | 最多 10 个 | 最多 10 个 | 最多 10 个 |
| 发布按钮 | `button.button-dhlUZE.primary-button` | `button.button-dhlUZE.primary-button` | `button.button-dhlUZE.primary-button` |

## 三、封面上传技术方案（核心难点）

### 3.1 问题背景

抖音文章发布**必须上传封面图**，否则无法发布。封面上传区域的 DOM 结构：

```html
<div class="mycard-c48v6G">
  <div class="addIcon-WtgoEN"><span class="addInnerIcon-KEMTi2">+</span></div>
  <span class="mycard-info-uBuPL9">点击上传封面图</span>
  <span class="mycard-info-text-YmWBDj">
    <span class="mycard-info-text-span-qfhmNK">选择封面</span>
  </span>
</div>
```

**关键问题**：
- 页面中**不存在持久化的** `<input type="file">` 元素
- 点击时动态创建 input 或使用 `showOpenFilePicker()` API
- 传统的 `DOM.setFileInputFiles` 方案找不到 input 节点
- 文件选择对话框是浏览器原生弹窗，CDP 无法直接操作

### 3.2 解决方案：JS 层 DataTransfer 直接注入 + CDP 兜底

采用**四层拦截机制**，完全在 JavaScript 层面设置文件，不弹出文件选择对话框：

```
┌─────────────────────────────────────────────────────────────────┐
│                     封面上传五层防护架构                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Layer 1: CDP Page.setInterceptFileChooserDialog         │   │
│  │ 浏览器层面拦截文件选择对话框，确保对话框不会弹出          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Layer 2: pointerdown/mousedown/click 捕获阶段拦截        │   │
│  │ 在事件捕获阶段调用 preventDefault() 阻止默认行为          │   │
│  │ 同时查找附近的 file input，立即通过 DataTransfer 设置文件  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Layer 3: HTMLInputElement.prototype.click() 拦截        │   │
│  │ 重写 input.click() 方法，代码调用时直接设置文件            │   │
│  │ 不调用原始方法，不弹出对话框                               │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Layer 4: MutationObserver 监听 DOM 变化                  │   │
│  │ 监听 document 变化，新出现的 file input 立即设置文件       │   │
│  │ 穿透 Shadow DOM 进行扫描                                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ Layer 5: showOpenFilePicker() 拦截 (File System Access) │   │
│  │ 重写 showOpenFilePicker，直接返回伪造的 FileHandle        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ CDP 兜底：3秒后 JS 方案未成功则降级为 DOM.getDocument    │   │
│  │ pierce:true 穿透所有 iframe/shadow DOM 查找 input         │   │
│  │ 使用 DOM.setFileInputFiles 设置文件                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 核心代码：buildSuperCoverPatchScript

**文件设置核心**：使用 `DataTransfer` API 构造 FileList，通过 `Object.defineProperty` 注入到 input：

```javascript
function setFileToInput(input) {
  // 1. 使用 DataTransfer 创建 FileList
  var dt = new DataTransfer();
  dt.items.add(coverFile);  // coverFile 是从 base64 构造的 File 对象
  var files = dt.files;
  
  // 2. 重定义 input.files 属性（三种方法兜底）
  try {
    Object.defineProperty(input, 'files', {
      value: files,
      writable: true,
      configurable: true,
    });
  } catch (e) {
    // 尝试原型链 setter
    var desc = Object.getOwnPropertyDescriptor(HTMLInputElement.prototype, 'files');
    if (desc && desc.set) desc.set.call(input, files);
  }
  
  // 3. 设置 value 属性
  Object.defineProperty(input, 'value', {
    value: 'C:\\fakepath\\' + coverFile.name,
    writable: true,
    configurable: true,
  });
  
  // 4. 按正确顺序触发事件（先 input 后 change，适配 React 合成事件）
  input.dispatchEvent(new Event('input', { bubbles: true }));
  input.dispatchEvent(new Event('change', { bubbles: true }));
}
```

### 3.4 base64 文件注入方案

封面图通过 Node.js 读取为 base64，注入到页面后在浏览器端构造 File 对象：

```javascript
// Node.js 侧
const fileBuffer = fs.readFileSync(coverFile);
const base64Data = fileBuffer.toString('base64');
const fileName = path.basename(coverFile);

// 注入到浏览器侧的 patch 脚本中
function base64ToFile(base64, filename, type) {
  var binaryStr = atob(base64);
  var bytes = new Uint8Array(binaryStr.length);
  for (var i = 0; i < binaryStr.length; i++) {
    bytes[i] = binaryStr.charCodeAt(i);
  }
  return new File([bytes], filename, { type: type });
}
```

### 3.5 跨 Frame 注入

patch 不仅注入主文档，还通过 `Page.createIsolatedWorld` 注入到所有 iframe：

```typescript
// 1. 获取 FrameTree
const frameTree = await win.webContents.debugger.sendCommand('Page.getFrameTree');

// 2. 遍历所有子 frame，创建隔离世界注入 patch
for (const childFrame of childFrames) {
  const isoRes = await win.webContents.debugger.sendCommand('Page.createIsolatedWorld', {
    frameId: childFrame.id,
    worldName: 'flowx-cover-' + Date.now(),
    grantUniversalAccess: true,
  });
  
  await win.webContents.debugger.sendCommand('Runtime.evaluate', {
    expression: superPatchScript,
    contextId: isoRes.executionContextId,
    returnByValue: true,
  });
}

// 3. 监听 Runtime.executionContextCreated，新 frame/context 创建时立即注入
```

### 3.6 弹窗关闭与"完成"按钮处理

封面上传后可能出现裁剪弹窗，需要点击"完成"按钮：

1. **点击前先发送 Escape 键**：关闭可能残留的文件选择对话框
2. **轮询点击"完成"按钮**：等待裁剪弹窗出现，重试10次（每次800ms）
3. **智能检测无弹窗情况**：如果检查到"编辑封面"文本或封面已有背景图，说明无裁剪弹窗，直接判定完成
4. **多次 Escape 兜底**：流程结束前再按3次 Escape 确保所有弹窗关闭

### 3.7 封面上传成功验证

三重验证机制：

```javascript
// 1. 检查封面区域是否有背景图
var coverImg = document.querySelector('.cover-Uudq5y');
if (coverImg) {
  var style = coverImg.getAttribute('style') || '';
  var hasBg = style.indexOf('background-image') !== -1 && style.indexOf('url(') !== -1;
  if (hasBg) return { hasCover: true };
}

// 2. 检查是否有"编辑封面"文本（上传成功后文本变化）
var allSpans = document.querySelectorAll('span');
for (var i = 0; i < allSpans.length; i++) {
  if ((allSpans[i].innerText || '').trim() === '编辑封面') {
    return { hasCover: true };
  }
}

// 3. 检查 mycard 文本变化（从"点击上传封面图"变为其他内容）
var mycard = document.querySelector('.mycard-c48v6G');
if (mycard) {
  var mycardText = (mycard.innerText || '').trim();
  if (mycardText.indexOf('点击上传封面图') === -1 && mycardText.length > 0) {
    return { hasCover: true };
  }
}
```

## 四、关键技术点

### 4.1 入口 URL 参数 `default-tab=5`

抖音创作者平台的上传页使用多个 tab：
- `default-tab=1` → 视频上传
- `default-tab=2` → 视频上传（另一种）
- `default-tab=3` → 图文发布
- `default-tab=4` → 合集发布
- `default-tab=5` → **文章（长文）发布**

不同 tab 下，页面会渲染不同的编辑器组件和按钮结构。`default-tab=5` 是文章发布的唯一入口。

### 4.2 引导页"我要发文"按钮

首次进入文章发布页时，可能会出现引导页：
```html
<button class="semi-button semi-button-primary container-drag-btn-hcUuRC">
  <span class="semi-button-content">我要发文</span>
</button>
```
需要检测并点击该按钮才能进入编辑器。

### 4.3 标题 30 字硬限制

抖音文章标题有严格的 30 字符限制：
```typescript
const ARTICLE_TITLE_MAX = 30;
```

### 4.4 正文 8000 字硬限制

抖音文章正文的平台限制是 8000 字符：
```typescript
const ARTICLE_CONTENT_MAX = 8000;
```

### 4.5 React 受控组件的输入方式

抖音文章编辑器的标题和正文字段都是 React 受控组件：
- 直接赋值 `el.value = text` → React 内部 state 不会更新 → 提交时内容被清空
- 必须使用 `document.execCommand('insertText')` 模拟原生用户输入 → 触发 React onChange

### 4.6 自动保存草稿的等待

抖音编辑器会**自动保存草稿**，填写完成后强制 `sleep(2000ms)`，给 React 提交和平台自动保存预留时间。

## 五、平台 meta 配置

`douyin.ts` 中的 meta 信息：

```typescript
export const meta: PlatformMeta = {
  contentLimits: {
    title: 80,      // 视频/图文：标题 80 字
    content: 1000,  // 视频/图文：正文 1000 字
  },
  articleLimits: {
    title: 30,      // 文章：标题 30 字（独立限制）
    content: 8000,  // 文章：正文 8000 字（独立限制）
  },
  contentTypes: ['video', 'image', 'article'],  // 支持三种发布模式
};
```

前端 `Publish.vue` 读取这个 meta，在用户切换到"文章"类型时：
- 标题输入框 maxlength 自动更新为 30
- 正文输入框 maxlength 自动更新为 8000
- 提示语更新为"抖音文章标题 30 字 / 正文 8000 字"
- 账号列表自动过滤掉不支持文章发布的平台（如快手）

## 六、与图文/视频发布的代码隔离

| 模块 | 共享 or 独立 | 说明 |
|------|-------------|------|
| `publishVideo` 方法 | 独立 | 完全独立的函数体，处理视频上传流程 |
| `publishImage` 方法 | 独立 | 完全独立的函数体，处理图文上传流程 |
| `publishArticle` 方法 | 独立 | 完全独立的函数体，处理文章多步发布流程（含封面上传） |
| `uploadArticleCover` 方法 | 独立 | 文章封面专用上传函数，仅被 `publishArticle` 调用 |
| `buildFillTitle` | 共享 | 通用标题填充，字数限制由调用者传入 |
| `buildFillContent` | 共享 | 通用正文填充，字数限制由调用者传入（null 表无限制） |
| `cdpClickPublishButton` | 共享 | CDP 穿透 shadow DOM 通用点击函数 |
| `waitForStable` / 导航跟踪 | 共享 | 所有平台和内容类型共享 |

> 架构原则：每个平台 × 每种内容类型 = 一个独立函数。函数之间只复用底层工具函数（fill, click, navigation-tracking），不共享任何业务流程控制。

## 七、封面上传 7 步流程（uploadArticleCover）

```
[1/5] 滚动到封面设置区域
      - scrollIntoView({ behavior: 'auto', block: 'center' })
      - 验证元素是否在视口内

[2/5] 注入文件上传 patch
      - Page.enable + Page.setInterceptFileChooserDialog（拦截原生对话框）
      - Runtime.enable + executionContextCreated 监听
      - 主文档注入 superPatchScript
      - 遍历所有 iframe，createIsolatedWorld 注入 patch
      - 等待 500ms 让 patch 生效

[3/5] 点击封面上传区域
      - CDP Input.dispatchMouseEvent 真实鼠标事件
      - 多点击点尝试（中心、左上、右下、上中、左中）
      - JS click() 兜底

[4/5] 等待文件设置（CDP兜底）
      - 轮询 40 次（每次 200ms），检查 __flowxCoverFileSet 标记
      - 第15次（约3秒）如果未成功，降级为 CDP DOM.getDocument(pierce:true)
      - 递归遍历所有节点（含 shadow DOM / iframe contentDocument）
      - 使用 DOM.setFileInputFiles 直接设置文件

[5/5] 等待上传并点击完成
      - 发送 Escape 键关闭可能的对话框
      - 轮询 10 次点击"完成"按钮
      - 智能检测无弹窗情况（"编辑封面"文本或背景图）
      - 最后按3次 Escape 确保弹窗关闭
      - 验证封面上传结果
```

## 九、已知限制

| 限制 | 说明 |
|------|------|
| 标题 30 字硬限制 | 由平台后端校验，前端截断无法绕过 |
| 正文 8000 字硬限制 | 由平台后端校验 |
| React 受控组件 | 必须用 execCommand 方式输入 |
| 封面图必填 | 抖音文章发布强制要求封面，不上传无法发布 |
| 封面裁剪弹窗 | 部分图片可能弹出裁剪弹窗，代码会自动点击"完成" |
| 自动保存草稿的异步 | 填写完成后需要 ~2 秒等待 |
