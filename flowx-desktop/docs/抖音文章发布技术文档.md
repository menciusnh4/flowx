# 抖音文章（长文）发布技术文档

> 发布入口：`https://creator.douyin.com/creator-micro/content/upload?default-tab=5`

## 一、文章发布 vs 图文发布 vs 视频发布

| 维度 | 视频发布 | 图文发布 | 文章发布 |
|------|---------|---------|---------|
| 入口 URL | 默认上传页 | `?default-tab=3` | `?default-tab=5` |
| 标题限制 | 55 字 | 80 字 | **30 字** |
| 正文限制 | —（标题即可发布） | 1000 字 | **8000 字** |
| 素材文件 | 视频（必填） | 图片（必填） | **无素材**（纯文字创作） |
| 话题标签 | 最多 10 个 | 最多 10 个 | 最多 10 个 |
| 发布按钮 | `button.button-dhlUZE.primary-button` | `button.button-dhlUZE.primary-button` | `button.button-dhlUZE.primary-button` |

## 二、核心流程

```
┌─────────────────────────────────────────────────────────────────┐
│  1) loadURL(https://creator.douyin.com/creator-micro/          │
│          content/upload?default-tab=5)                         │
│     ↓                                                          │
│  2) 等待页面加载稳定（waitForStable）                           │
│     ↓                                                          │
│  3) 填写标题（30 字限制）                                       │
│     buildFillTitle(title, ARTICLE_TITLE_MAX=30)               │
│     - 抖音标题使用 React 受控组件（contenteditable）           │
│     - 🔑 使用 execCommand('insertText') 触发 onChange           │
│     - 超过 30 字自动截断                                        │
│     ↓                                                          │
│  4) 填写正文（8000 字限制，富文本编辑器）                        │
│     buildFillContent(content, ARTICLE_CONTENT_MAX=8000)       │
│     - 正文编辑器检测：article-content / editor / editor-text  │
│     - 自动分段写入（每 1000 字左右自然换行）                    │
│     - 超过 8000 字自动截断                                      │
│     ↓                                                          │
│  5) 填写话题（可选，最多 10 个）                                │
│     appendTagsToContent(tags)                                  │
│     ↓                                                          │
│  6) 等待保存草稿（抖音自动保存草稿，正文较多时需等待）           │
│     sleep(2000ms)                                              │
│     ↓                                                          │
│  7) 点击"发布"按钮                                              │
│     优先级:                                                      │
│       1) Class 选择器: button.button-dhlUZE.primary-button     │
│       2) 文本匹配: "发布" / "发布作品" / "发布文章"             │
│       3) 降级为 CDP 穿透 shadow DOM 点击                        │
│     ↓                                                          │
│  8) 等待发布结果（60 秒超时）                                    │
│     检测 URL: 离开 upload 页 或 出现 "发布成功/已发布" 文本      │
│     ↓                                                          │
│  9) 完成                                                       │
└─────────────────────────────────────────────────────────────────┘
```

## 三、关键技术点

### 3.1 入口 URL 参数 `default-tab=5`

抖音创作者平台的上传页使用多个 tab：
- `default-tab=1` → 视频上传
- `default-tab=2` → 视频上传（另一种）
- `default-tab=3` → 图文发布
- `default-tab=4` → 合集发布
- `default-tab=5` → **文章（长文）发布**

不同 tab 下，页面会渲染不同的编辑器组件和按钮结构。`default-tab=5` 是文章发布的唯一入口。

### 3.2 标题 30 字硬限制

抖音文章标题有严格的 30 字符限制：
```typescript
const ARTICLE_TITLE_MAX = 30;
// 在 publishArticle 中，传递给 buildFillTitle 时强制限制
// 超过 30 字 → truncate + warn log
```

### 3.3 正文 8000 字硬限制

抖音文章正文的平台限制是 8000 字符：
```typescript
const ARTICLE_CONTENT_MAX = 8000;
// 在 publishArticle 中，传递给 buildFillContent
// 超过 8000 字 → truncate + warn log
```

8000 字大约对应 4-6 篇普通博文的长度，足够绝大多数长文场景。

### 3.4 React 受控组件的输入方式

抖音文章编辑器的标题和正文字段都是 React 受控组件：
- 直接赋值 `el.value = text` → React 内部 state 不会更新 → 提交时内容被清空
- 必须使用 `document.execCommand('insertText')` 模拟原生用户输入 → 触发 React onChange

具体实现与图文发布相同，详见 `抖音图文发布技术文档.md` 第 4 节。

### 3.5 自动保存草稿的等待

抖音编辑器会**自动保存草稿**（localStorage / API 调用），当正文较长时（>1000 字），自动保存会占用几秒的异步处理时间。

如果在自动保存完成前立即点击"发布"，可能导致：
- 发布按钮还未激活（灰态）
- 内容被部分截断

**处理策略**：填写完成后强制 `sleep(2000ms)`，给 React 提交和平台自动保存预留时间。

### 3.6 发布按钮的点击流程

抖音文章发布的"发布"按钮与图文发布的按钮**是同一条 DOM 路径**：

```html
<button class="button-dhlUZE primary-button">发布</button>
```

点击策略与图文发布共享：

1. **Class 直接命中**：`querySelector('button.primary-button')` → `click()`
2. **文本模糊匹配**：遍历所有 `<button>`，匹配"发布/发布作品/发布文章" → `click()`
3. **CDP 兜底**：JS click 失败时，通过 `Input.dispatchMouseEvent` 合成真实鼠标点击

### 3.7 发布结果检测

文章发布后，页面行为与视频/图文发布类似：
- URL 离开 `/content/upload` → 视为成功
- body 文本中出现"发布成功"、"已发布"、"发布完成" → 视为成功
- body 文本中出现"发布失败"、"内容违规"、"网络异常" → 视为失败

## 四、平台 meta 配置

`douyin.ts` 中的 meta 信息：

```typescript
export const meta: PlatformMeta = {
  contentLimits: {
    title: 80,      // 视频/图文：标题 80 字
    content: 1000,  // 视频/图文：正文 1000 字
  },
  articleLimits: {
    title: 30,      // 文章：标题 30 字（独立限制）
    content: 8000,  // 文章：正文 8000 字（独立限制）
  },
  contentTypes: ['video', 'image', 'article'],  // 支持三种发布模式
};
```

前端 `Publish.vue` 读取这个 meta，在用户切换到"文章"类型时：
- 标题输入框 maxlength 自动更新为 30
- 正文输入框 maxlength 自动更新为 8000
- 提示语更新为"抖音文章标题 30 字 / 正文 8000 字"

## 五、与图文/视频发布的代码隔离

| 模块 | 共享 or 独立 | 说明 |
|------|-------------|------|
| `publishVideo` 方法 | 独立 | 完全独立的函数体，处理视频上传流程 |
| `publishImage` 方法 | 独立 | 完全独立的函数体，处理图文上传流程 |
| `publishArticle` 方法 | 独立 | 完全独立的函数体，处理文章多步发布流程 |
| `buildFillTitle` | 共享 | 通用标题填充，字数限制由调用者传入 |
| `buildFillContent` | 共享 | 通用正文填充，字数限制由调用者传入（null 表无限制） |
| `cdpClickPublishButton` | 共享 | CDP 穿透 shadow DOM 通用点击函数 |
| `waitForStable` / 导航跟踪 | 共享 | 所有平台和内容类型共享 |

> 架构原则：每个平台 × 每种内容类型 = 一个独立函数。函数之间只复用底层工具函数（fill, click, navigation-tracking），不共享任何业务流程控制。

## 六、已知限制

| 限制 | 说明 |
|------|------|
| 标题 30 字硬限制 | 由平台后端校验，前端截断无法绕过 |
| 正文 8000 字硬限制 | 由平台后端校验 |
| React 受控组件 | 必须用 execCommand 方式输入 |
| 自动保存草稿的异步 | 填写完成后需要 ~2 秒等待 |
| 不支持封面图 | 文章发布模式下当前未提供封面图上传接口 |
