## 一、页面结构与关键元素

### 1.1 发布页面入口

知乎图文发布基于**"发想法"**功能实现，发布页面为创作者中心：`https://www.zhihu.com/creator`

知乎没有专门的图文发布入口，而是通过"分享此刻的想法"功能发布图文内容。

### 1.2 编辑器两种状态

知乎想法编辑器有**折叠**和**展开**两种状态：

#### 折叠状态（初始状态）

```html
<!-- 折叠状态：只有一行提示文本 -->
<div class="css-1lkz3hi">分享此刻的想法...</div>
```

- 视觉上只有一行提示文本
- 点击后展开完整编辑表单
- DOM 中 `WritePinV2-Form` 元素可能已存在但不可见

#### 展开状态（激活后）

```html
<div class="WritePinV2-Form css-1a7oo9s">
  <!-- 标题区域 -->
  <div class="TitleArea css-1dzflou">
    <textarea placeholder="标题" name="title" class="css-1ta9b1n"></textarea>
  </div>

  <!-- 正文编辑区（Draft.js） -->
  <div class="EditorArea css-tgh5h8">
    <div class="InputLike css-18i781m Editable">
      <div class="Dropzone Editable-content RichText RichText--editable ...">
        <div class="DraftEditor-root">
          <!-- 占位符 -->
          <div class="public-DraftEditorPlaceholder-root">
            <div class="public-DraftEditorPlaceholder-inner">分享你此刻的想法...</div>
          </div>
          <!-- 编辑器内容 -->
          <div class="DraftEditor-editorContainer">
            <div class="public-DraftEditor-content" contenteditable="true" role="textbox">
              <!-- Draft.js 内容... -->
            </div>
          </div>
        </div>
      </div>
    </div>
  </div>

  <!-- 图片区域（独立于编辑器） -->
  <div class="ImageArea css-1g0jrq7">
    <div class="DraggableTags css-2imjyh">
      <!-- 每张图片一个 tag -->
      <div class="DraggableTags-tag hotspot-xxxxxx">
        <div class="DraggableTags-tag-drag">
          <div class="css-1jm9d8k">
            <img src="https://pic-private.zhihu.com/..." alt="image" class="css-694y47">
            <div class="image-mask css-1y1kk8p">
              <div class="css-18qox9q">编辑</div>
              <div class="css-2tuzlq">（删除按钮）</div>
            </div>
          </div>
        </div>
      </div>
      <!-- 添加图片按钮 -->
      <div class="DraggableTags-tag DraggableTags-undraggable hotspot-xxxxxx">
        <div class="DraggableTags-tag-drag">

    </div>

```

**关键元素：**

| 元素 | 选择器 | 说明 |
|:---|:---|:---|

### 1.3 字数与内容限制

| 字段 | 限制 | 元素 |
|:---|:---|:---|
| 标题 | 50 个字符 | `textarea[name="title"]` |
| 正文 | 1000 个字符 | `.public-DraftEditor-content`（Draft.js） |
| 图片 | 最多 18 张 | `.ImageArea .DraggableTags-tag` |
| 话题 | 嵌入正文，无单独限制 | `#话题名#` 格式 |

```html
          <!-- 上传中状态：按钮不可点击 -->

          <!-- 上传完成状态：按钮可点击 + 计数信息 -->

            </div>

          -->
        </div>
      </div>
    </div>
  </div>

```

### 2.2 按钮状态变化

| 状态 | 按钮 class | 说明 |
|:---|:---|:---|
| 上传中 | `css-12pk5sz` | 按钮不可点击（disabled） |
| 上传完成 | `css-owamhi` | 按钮可点击，可插入图片 |

### 2.3 上传完成判断标志

1. **按钮 class 变化**：从 `css-12pk5sz` 变为 `css-owamhi`
2. **"已上传"文本**：出现 `css-yihm2v` 容器，包含"已上传 X 张图片"文本
3. **计数信息**：出现 `css-tl1hby` / `css-vurnku` 等计数元素

## 三、完整发布流程

知乎图文发布共 **10 步**，全程使用 `BrowserWindow` + CDP 实现：

```
步骤 1: 创建独立窗口（账号隔离 partition）
步骤 2: 加载 https://www.zhihu.com/creator
步骤 3: 导航跟踪器等待页面稳定
步骤 4: 登录态检测（z_c0 cookie + DOM 元素双重验证）
步骤 5: 激活编辑器（检测 css-1lkz3hi 可见性 → 点击展开表单）
步骤 6: 填写标题（50字限制，nativeSetter + input 事件）
步骤 7: 上传图片（点击图片按钮 → Modal弹窗 → CDP注入 → 等待上传完成 → 点击插入）
步骤 8: 填写正文 + 话题（1000字限制，Draft.js + 多层方案兜底）
步骤 9: 点击发布按钮（WritePinToolbar 右侧发布按钮）
步骤 10: 等待发布结果（URL变化 + 成功文本轮询）
```

### 3.1 步骤详解

#### 步骤 1-4：窗口创建与页面加载

与知乎视频发布一致：
- 使用 `persist:account_{id}` 作为 `partition`，实现账号间完全隔离
- 通过 `NavigationTracker` 等待页面静默（1.5s 无新请求），确保 DOM 稳定
- 登录态检测：`z_c0` cookie + 页面元素双重验证

#### 步骤 5：激活编辑器（关键步骤）

**核心判断逻辑**：通过 `css-1lkz3hi` 提示文本的**可见性**判断编辑器状态，而非 DOM 元素是否存在。

```javascript
// 错误方式：只检查 DOM 元素是否存在（WritePinV2-Form 始终存在）
var form = document.querySelector('.WritePinV2-Form'); // ❌ 不可靠

// 正确方式：检查提示文本的可见性
var hintEl = document.querySelector('.css-1lkz3hi');
var hintVisible = hintEl && getComputedStyle(hintEl).display !== 'none'
                  && hintEl.offsetParent !== null;

if (hintVisible) {
  // 编辑器未激活 → 点击激活
  hintEl.click();
} else {
  // 编辑器已激活
}
```

**多层可见性检测**：
1. `getComputedStyle(el).display !== 'none'`
2. `getComputedStyle(el).visibility !== 'hidden'`
3. `el.offsetParent !== null`（元素在布局中）

**激活方式（按优先级）**：
1. 直接点击 `.css-1lkz3hi` 元素
2. 向上遍历父元素，点击可点击的父容器
3. 兜底：`.css-1tdhe7b` 容器
4. 文本匹配：包含"分享此刻的想法"的可见元素

#### 步骤 6：标题填写

与知乎视频发布一致，使用 `nativeSetter` + `input/change` 事件：

```typescript
function buildFillPinTitleScript(title: string): string {
  return `
    (function() {
      var ta = document.querySelector('.WritePinV2-Form textarea[name="title"]');
      if (!ta) return { ok: false, error: 'no-title-textarea' };
      
      var maxLen = 50;
      var text = ${JSON.stringify(title)}.slice(0, maxLen);
      
      var nativeSetter = Object.getOwnPropertyDescriptor(
        window.HTMLTextAreaElement.prototype, 'value'
      ).set;
      nativeSetter.call(ta, text);
      ta.dispatchEvent(new Event('input', { bubbles: true }));
      ta.dispatchEvent(new Event('change', { bubbles: true }));
      
      return { ok: true, length: text.length };
    })();
  `;
}
```

#### 步骤 7：图片上传（复杂步骤）

图片上传分为 5 个子步骤：

```
点击图片按钮 → 等待 Modal 出现 → CDP 注入文件 → 等待上传完成 → 点击插入
```

##### 7.1 点击图片按钮

点击 `.WritePinToolbar .ZDI--Image24` 或父级 button 元素，触发上传弹窗。

**遮罩层保护**：注入脚本阻止 Modal 遮罩层的点击事件冒泡，防止误点遮罩关闭弹窗。

##### 7.2 等待 Modal 出现

轮询检测 Modal 是否可见（每 300ms 检查，最多 5 秒）：
- 检测 `.Modal` 元素可见性
- 验证内部包含"本地图片"、"上传图片"、"插入图片"或"已上传"文本
- 或检测 `.css-1v0c8fj` 按钮容器

##### 7.3 CDP 注入文件

**双方案获取 file input 的 nodeId**：

**方案 1（推荐）**：JS 定位 + `DOM.querySelector`
```typescript
// 步骤 1：JS 脚本定位 Modal 内的 file input，并添加标记
const findScript = `
  var input = targetModal.querySelector('input[type="file"][accept*="image"]');
  input.setAttribute('data-upload-target', 'zhihu-pin-image');
  return { selector: 'input[data-upload-target="zhihu-pin-image"]' };
`;

// 步骤 2：CDP DOM.querySelector 获取 nodeId
const nodeId = await debugger.sendCommand('DOM.querySelector', {
  nodeId: docNode.root.nodeId,
  selector: 'input[data-upload-target="zhihu-pin-image"]',
});
```

**方案 2（兜底）**：`DOM.getDocument` 全量遍历
```typescript
// 深度遍历所有节点（含 shadow DOM）
const doc = await debugger.sendCommand('DOM.getDocument', { depth: -1, pierce: true });
// 扁平化所有节点，找到 type=file 的 input
// 建立 parentId 映射，验证是否在 Modal 内
```

**注入文件**：
```typescript
await debugger.sendCommand('DOM.setFileInputFiles', {
  nodeId: targetNodeId,
  files: [imagePath],
});
```

**事件触发**：注入后触发 `change`、`input`、`focus/blur` 等事件，确保 React 感知到文件变化。

##### 7.4 等待上传完成

轮询检测上传状态（每秒检查，最多 30 秒）：

**上传完成标志（满足任一即可）**：
1. 插入按钮 class 为 `css-owamhi`（可点击状态）
2. 出现 `css-yihm2v` 元素，包含"已上传"文本
3. 按钮 `disabled` 属性为 false 且有预览图

**日志输出**：实时输出上传状态（上传中/有预览图/按钮可点击等）

##### 7.5 点击插入按钮

**点击优先级**：
1. `button.css-owamhi`（上传完成状态的按钮，最精确）
2. 按文本匹配：包含"插入"、"确认"、"确定"的 button
3. `role="button"` 的 div 元素

**点击方式**：
- 优先 `.click()` 方法
- 兜底：`MouseEvent` 事件派发

##### 7.6 等待图片出现在编辑器

点击插入后，等待图片出现在 `ImageArea` 区域（每 1 秒检查，最多 25 秒）。

**图片检测方式**：
1. 优先检测 `.ImageArea .DraggableTags-tag` 内的真实图片（排除 plus 按钮）
2. 兜底：检测 `.ImageArea` 内的真实图片 URL（排除 ZDI 图标）
3. 补充：检测编辑器内的图片（atomic block、figure 等）

#### 步骤 8：正文填写（Draft.js）

知乎正文使用 Draft.js 编辑器，简单 DOM 操作不会更新 React state。采用**四层兜底方案**：

##### 方案 1：paste 事件（推荐）

```javascript
// 构造 ClipboardEvent 粘贴文本
var dataTransfer = new DataTransfer();
dataTransfer.setData('text/plain', fullText);

var pasteEvent = new ClipboardEvent('paste', {
  bubbles: true,
  cancelable: true,
  clipboardData: dataTransfer,
});
editor.dispatchEvent(pasteEvent);
```

- paste 事件是 Draft.js 原生支持的输入方式
- 会正确触发 Draft.js 的 `onChange`，更新 EditorState

##### 方案 2：beforeinput + input 事件序列

逐个字符触发标准输入事件：
```javascript
for (var ci = 0; ci < fullText.length; ci++) {
  editor.dispatchEvent(new InputEvent('beforeinput', {
    bubbles: true, cancelable: true, inputType: 'insertText', data: ch,
  }));
  editor.dispatchEvent(new InputEvent('input', {
    bubbles: true, cancelable: false, inputType: 'insertText', data: ch,
  }));
}
```

##### 方案 3：document.execCommand('insertText')

```javascript
editor.focus();
document.execCommand('selectAll', false, null);
document.execCommand('delete', false, null);
document.execCommand('insertText', false, fullText);
```

##### 方案 4：CDP 键盘输入兜底（最可靠）

如果 JS 方案全部失败，使用 Chrome DevTools Protocol 模拟真实键盘输入：

```typescript
for (let ci = 0; ci < fullText.length; ci++) {
  await win.webContents.debugger.sendCommand('Input.dispatchKeyEvent', {
    type: 'char',
    text: fullText[ci],
  });
  if (ci % 20 === 0) await sleep(50); // 每 20 字符停顿
}
```

- 最接近真实用户行为
- 100% 触发 React state 更新
- 速度稍慢（每字符一次 CDP 调用）

##### 话题处理

话题通过 `#话题名#` 格式直接追加到正文末尾，受 1000 字符总长度限制：

```javascript
var tagStr = '';
for (var ti = 0; ti < tags.length; ti++) {
  var tagText = '#' + tags[ti] + ' ';
  if (baseText.length + tagStr.length + tagText.length <= 1000) {
    tagStr += tagText;
  }
}
var fullText = (baseText + tagStr).slice(0, 1000);
```

#### 步骤 9：点击发布按钮

发布按钮位于 `.WritePinToolbar-RightGroup` 内。

**选择策略**：
1. 精确匹配：`.WritePinToolbar-RightGroup button`（最后一个可见按钮）
2. 文本匹配：包含"发布"的按钮
3. 类名匹配：包含 `primary`、`submit` 等关键词

#### 步骤 10：发布结果检测

轮询检测（最长 180 秒），成功判断条件：
1. **URL 变化**：页面跳转到想法详情页或管理页
2. **成功文本**：页面出现"发布成功"、"已发布"、"提交成功"等文本
3. **表单消失**：`.WritePinV2-Form` 不再存在

## 四、核心技术难点

### 4.1 编辑器激活状态判断

**问题**：知乎想法编辑器有折叠/展开两种状态，DOM 元素始终存在但视觉上折叠。仅检测 DOM 元素存在性会误判为"已激活"，跳过点击步骤。

**解决方案**：通过 `css-1lkz3hi` 提示文本的**可见性**判断状态：
- `css-1lkz3hi` 可见 → 编辑器未激活 → 需要点击
- `css-1lkz3hi` 不可见 + `ImageArea` 可见 → 编辑器已激活

**相关代码**：
- `buildActivatePinEditorScript()` — 激活编辑器
- `buildPinEditorReadyProbe()` — 检测编辑器是否就绪

### 4.2 图片上传 Modal 的动态渲染与 nodeId 获取

**问题**：点击图片按钮后 Modal 才动态创建，CDP 注入需要精确的 `nodeId`，且 Modal 内元素动态渲染。

**解决方案**：
1. **双方案获取 nodeId**：
   - 方案1：JS 定位 + `DOM.querySelector`（先加标记再查询）
   - 方案2：`DOM.getDocument(depth:-1, pierce:true)` 全量遍历兜底
2. **遮罩层保护**：阻止遮罩层点击事件冒泡，防止误关弹窗
3. **按钮状态检测**：通过 `css-12pk5sz`（上传中）→ `css-owamhi`（完成）判断上传进度

**相关代码**：
- `uploadImageInModal()` — Modal 内图片上传主函数
- `buildImageUploadModalProbe()` — Modal 可见性检测
- `buildModalImageUploadedProbe()` — 上传完成状态检测

### 4.3 图片区域识别（非编辑器内图片）

**问题**：知乎图文的图片不嵌入 Draft.js 编辑器，而是在独立的 `ImageArea` / `DraggableTags` 区域。在编辑器内查找图片永远找不到，导致误判上传失败。

**解决方案**：
- 优先检测 `ImageArea` 内的 `DraggableTags-tag` 数量
- 排除 plus 按钮和 ZDI 图标等非真实图片
- 测试模式检测逻辑同步更新

**相关代码**：
- `buildImageInsertedProbe()` — 图片插入检测
- `buildPinTestModeProbeScript()` — 测试模式检测

### 4.4 Draft.js 正文填写（React 状态同步）

**问题**：Draft.js 有自己的内部数据模型（EditorState），直接修改 DOM 不会更新内部状态，React 重渲染时会丢失。

**解决方案**：四层兜底方案
1. `ClipboardEvent('paste')` — 粘贴事件，Draft.js 原生支持
2. 逐个字符 `beforeinput + input` 事件序列
3. `document.execCommand('insertText')`
4. CDP `Input.dispatchKeyEvent` 模拟真实键盘输入（最可靠）

**相关代码**：
- `buildFillPinContentScript()` — 正文填写脚本（前三种方案）
- CDP 键盘输入（正文填写调用处的兜底逻辑）

## 五、测试模式

支持 `testMode` 参数，用于调试表单填写效果而不真正发布：

```typescript
const request: PublishRequest = {
  platform: 'zhihu',
  contentType: 'image',
  testMode: true,  // 只填写表单，不点击发布
  // ...其他参数
};
```

测试模式会检测以下字段：

| 字段 | 类型 | 检测方式 |
|:---|:---|:---|
| 标题 | textarea | `.WritePinV2-Form textarea[name="title"]` 的 value |
| 正文 | contenteditable | `.public-DraftEditor-content` 的 innerText / textContent |
| 图片 | ImageArea | `.ImageArea .DraggableTags-tag` 内的真实图片数量 |
| 话题 | 嵌入正文 | 从正文中提取 `#话题名#` 格式 |
| 发布按钮 | button | `.WritePinToolbar-RightGroup` 内的可见按钮 |

结果通过 `testResult` 返回，包含每个字段的填充状态和发布按钮信息。测试模式下窗口保持打开，方便人工检查。

## 六、代码隔离与架构模式

### 6.1 平台独立实现

知乎的所有发布逻辑（视频 + 图文）完全独立在 `zhihu.ts` 中，不与其他平台共享发布流程代码。共享的只有 `shared.ts` 中的基础工具（窗口管理、导航跟踪、CDP 上传等）。

### 6.2 功能函数列表

| 函数 | 用途 |
|:---|:---|
| `publishZhihuImage()` | 图文发布主入口 |
| `buildActivatePinEditorScript()` | 激活编辑器（点击提示文本） |
| `buildPinEditorReadyProbe()` | 检测编辑器是否就绪 |
| `buildFillPinTitleScript()` | 填写标题 |
| `buildFillPinContentScript()` | 填写正文（含话题） |
| `uploadImageInModal()` | Modal 内上传图片（CDP注入） |
| `buildImageUploadModalProbe()` | 检测图片上传弹窗是否可见 |
| `buildModalImageUploadedProbe()` | 检测图片是否上传完成 |
| `buildClickInsertInModalScript()` | 点击弹窗内的插入按钮 |
| `buildImageInsertedProbe()` | 检测图片是否已插入编辑器 |
| `buildPinTestModeProbeScript()` | 测试模式检测脚本 |

### 6.3 两级分发

`PlatformDispatcher.createExecutor('zhihu', 'image')` → 分发到 `zhihu.ts` 的 `publishZhihuImage` 函数。

## 七、排错指南

### 7.1 常见问题

| 现象 | 可能原因 | 排查方法 |
|:---|:---|:---|
| 正文填了但看不到 | Draft.js state 未同步 | 检查日志中 method 字段，看使用了哪种方案 |
| 图片上传了但检测不到 | 在编辑器内找图片，而不是 ImageArea | 检查 imageAreaFound 和 draggableTagCount 日志 |
| 编辑器没展开就开始填写 | 激活检测失效（DOM存在但不可见） | 检查 hintVisible 日志，确认是否正确检测 |
| 上传弹窗出现又关闭 | 点击了遮罩层，或重试时 toggle 了 | 检查遮罩层保护是否生效，ensureModalOpen 逻辑 |
| 注入文件失败 | file input 的 nodeId 获取失败 | 检查两种方案的日志，看哪种失败 |
| 上传按钮一直不可点击 | 上传超时或网络问题 | 检查 uploadStatus 日志，确认上传进度 |

### 7.2 调试技巧

1. **打开 DevTools**：在发布窗口按 F12 或设置 `win.webContents.openDevTools()`
2. **查看日志**：控制台输出每一步的详细日志，包括方案选择和返回结果
3. **测试模式**：先用 `testMode: true` 验证表单填写，再测真实发布
4. **检查激活状态**：在 DevTools Console 中执行：
   ```javascript
   var hint = document.querySelector('.css-1lkz3hi');
   console.log('hint visible:', hint && hint.offsetParent !== null);
   ```
5. **检查图片区域**：
   ```javascript
   var imgArea = document.querySelector('.ImageArea');
   console.log('tags:', imgArea.querySelectorAll('.DraggableTags-tag').length);
   ```
6. **检查按钮状态**：
   ```javascript
   console.log('uploading btn:', document.querySelector('button.css-12pk5sz'));
   console.log('done btn:', document.querySelector('button.css-owamhi'));
   ```
