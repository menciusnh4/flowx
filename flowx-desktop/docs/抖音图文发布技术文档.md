## 一、目标页面结构分析

### 1.1 发布页 URL（图文专用）

```
图文发布 URL: https://creator.douyin.com/creator-micro/content/upload?default-tab=3
                                                          ↑
                                                    default-tab=3 = 图文模式
                                                    default-tab=1 = 视频模式（默认）
```

**为什么用 default-tab=3**：
- 抖音创作者中心的发布页支持多模式切换
- URL 参数 `default-tab=3` 直接进入图文发布模式
- 省去了点击切换 tab 的步骤，降低失败概率
- 这是图文发布的专用入口

### 1.2 页面核心 DOM 结构

#### 文件上传区域

```html
<!-- 图文上传区域（default-tab=3 时自动激活） -->
<div class="upload-container">
  <input type="file" accept="image/png, image/jpeg, image/webp" multiple />
  <div class="upload-zone">
    <span>点击或拖拽图片到此处上传</span>
  </div>
</div>
```

**关键点**：
- `uploadViaCDP(win, files, log, contentType='image')` 会自动选择 accept 含 `image/` 的 input
- 支持多图片上传
- 上传成功后自动进入内容编辑页面（SPA 路由跳转，需要导航跟踪器处理）

#### 标题输入框

```html
<!-- 标题输入框（React 受控组件） -->
<input
  type="text"
  class="input-title"
  placeholder="添加作品标题"
/>
```

**React 受控组件的特殊性**：见第三节「React 受控组件兼容」详细说明。

#### 正文/描述区域

```html
<!-- 描述输入框（contenteditable div） -->
<div contenteditable="true" class="textarea-container" data-placeholder="添加作品描述">
  <!-- 用户输入的文本内容 -->
</div>
```

抖音图文发布的正文区域是 contenteditable div，不是 ProseMirror 富文本编辑器（与小红书不同）。

#### 发布按钮

```html
<!-- 发布按钮（标准 DOM button） -->
<button class="button-dhlUZE primary-cECiOJ fixed-J9O8Yw">
  发布
</button>
```

**关键选择器**：
- `button.button-dhlUZE.primary-cECiOJ` — 最精确匹配
- `button.primary-cECiOJ` — 兜底选择器
- `button:contains('发布')` — 文本匹配兜底

发布按钮是标准 DOM button，**不在 shadow DOM 内**，JS click() 即可生效（与小红书不同）。

## 二、完整发布流程

### 2.1 PlatformDispatcher 分发

```
PublishEngine.submit(req)
  └─> PlatformDispatcher.createExecutor('douyin', 'image')
        └─> douyin.publishImage(accountId, req, onProgress)
              └─> 完全独立的图文发布流程（与 publishVideo 互不影响）
```

### 2.2 发布流程图

```
1. 窗口初始化
   └─> makePublishWindow(accountId)
         · BrowserWindow（sandbox + contextIsolation + partition）
         · 监听 render-process-gone
   └─> attachNavigationTracker(win, log)
         · 导航跟踪器（处理 SPA 跳转）

2. 导航到图文发布页
   └─> win.loadURL('https://creator.douyin.com/creator-micro/content/upload?default-tab=3')
   └─> tracker.waitForStable(1500, 15000)  ← 1.5s 静默期

3. 登录态检测
   └─> evalJS(win, buildDetectLoggedInScript(), '检测登录态')
         · 检查 URL 不包含 login/signin/redirect
         · 检查 body.innerText 命中 "创作中心"/"内容管理" 等关键词
         · 检查存在 ".header-name" / ".user-info-name" 等账号元素

4. 文件上传（CDP）
   └─> uploadViaCDP(win, files, log, contentType='image')
         · CDP DOM.getDocument({ depth: -1 }) 深度遍历
         · 按 accept 属性评分排序（含 image/ 的优先）
         · DOM.setFileInputFiles(nodeId, files) 注入文件
         · 派发 change + input 事件
   └─> sleep(1500) 等待上传完成
   └─> waitForUploadComplete(win, log) 轮询检测

5. 填写标题（React 受控组件）
   └─> evalJS(win, buildFillTitleScript(title), '填写标题')
         · placeholder 含 "标题"/"作品标题" 的 input
         · execCommand('selectAll') → delete → insertText
         · 确保 React state 更新

6. 填写正文/描述 + 话题
   └─> evalJS(win, buildFillContentScript(content, tags), '填写描述')
         · contenteditable div（含 data-placeholder="添加作品描述"）
         · 话题：转换为 #话题1 #话题2 行文本，附加在内容末尾
         · 最多 1000 字，超过自动 truncate

7. 点击发布按钮
   └─> evalJS(win, buildClickPublishScript(), '点击发布')
         · 查找 button.button-dhlUZE.primary-cECiOJ
         · 或查找含 "发布" 文本的 button 元素
         · element.click() — 标准 DOM click，正常生效
   └─> sleep(1500) 等待点击生效

8. 发布结果检测
   └─> sleep(3000) 等待页面跳转
   └─> evalJS(win, verifyScript, '验证发布结果')
         · 成功：URL 跳转到 /content/manage，或 body 含 "发布成功"
         · 失败：停留在 /upload，或 body 含错误信息

9. 完成/清理
   └─> 成功：win.destroy() 自动关闭
   └─> 失败：保持窗口打开，让用户查看/手动发布
```

## 三、核心难点与解决方案

### 难点 1：React 受控组件标题填写

**问题**：抖音标题输入框是 React `useState` 受控组件。

```javascript
// React 概念代码：
const [title, setTitle] = useState('');

<input
  type="text"
  value={title}                   // ← 渲染值与 state 同步
  onChange={e => setTitle(e.target.value)}  // ← 仅通过 onChange 更新 state
/>
```

**时序问题**（与小红书相同，但抖音更明显）：
```
t1: JS 设置 input.value = '我的标题'  ← 改了 DOM，没改 state
t2: React 重渲染（其他字段更新时触发  ← React 检测到 state 与 DOM 不一致
t3: input.value = ''                   ← React 将 input.value 重置为 state.title('')
                                                           ↑ 标题被清空！
```

**解决方案：`document.execCommand('insertText')`

```javascript
// ✅ buildFillTitleScript 内部实现（douyin.ts）
input.focus();
try { document.execCommand('selectAll'); } catch(e) {}
try { document.execCommand('delete'); } catch(e) {}
document.execCommand('insertText', false, text);
// → execCommand 触发原生 input 事件
// → React 监听该事件，调用 onChange(e)
// → setTitle(text) 更新 state
// → 下次重渲染时 value 与 state 一致
```

**为什么 execCommand 能触发 onChange**：
- `document.execCommand('insertText')` 是浏览器原生命令
- 浏览器会在内容变化时派发原生 `input` 事件
- React 的 onChange 监听的就是原生 input 事件
- 因此 `e.target.value` 是最新值，`setTitle()` 能正确更新

**完整的 buildFillTitleScript 代码**：

```javascript
(function() {
  // 1. 查找标题输入框
  //    优先级：placeholder 含 "标题"/"作品标题" → input.type="text" →
  var inputs = document.querySelectorAll('input[type="text"]');
  var target = null;
  for (var i = 0; i < inputs.length; i++) {
    var inp = inputs[i];
    var ph = (inp.getAttribute && inp.getAttribute('placeholder')) || '';
    if (/标题|作品标题|添加标题/i.test(ph)) {
      target = inp;
      break;
    }
  }
  if (!target) {
    // 兜底：第一个 textarea（有的版本用 textarea）
    var textareas = document.querySelectorAll('textarea');
    for (var j = 0; j < textareas.length; j++) {
      var ta = textareas[j];
      var ph2 = (ta.getAttribute && ta.getAttribute('placeholder')) || '';
      if (/标题|作品标题/i.test(ph2)) {
        target = ta;
        break;
      }
    }
  }
  if (!target) {
    // 最后兜底：第一个 input[type=text]
    if (inputs.length > 0) target = inputs[0];
  }
  if (!target) return { ok: false, reason: 'no-input' };

  // 2. 字数限制（标题最多 80 字）
  var text = "用户输入的标题";
  if (text.length > 80) {
    text = text.slice(0, 79) + '\u2026';
  }

  // 3. execCommand 模拟原生输入
  target.focus();
  try { document.execCommand('selectAll'); } catch(e) {}
  try { document.execCommand('delete'); } catch(e) {}
  try {
    document.execCommand('insertText', false, text);
  } catch(eCmd) {
    // 兜底：直接赋值 + dispatchEvent（部分浏览器不支持 execCommand）
    target.value = text;
    try { target.dispatchEvent(new Event('input', { bubbles: true })); } catch(eEv) {}
    try { target.dispatchEvent(new Event('change', { bubbles: true })); } catch(eEv2) {}
  }
  return { ok: true, method: 'input-execcommand' };
})()
```

### 难点 2：正文描述填写（contenteditable div）

抖音描述区域是普通的 contenteditable div，不是 ProseMirror，处理方式相对简单：

```javascript
(function() {
  // 1. 查找描述区域（contenteditable div，含 data-placeholder 提示）
  var editables = document.querySelectorAll('[contenteditable="true"]');
  var target = null;
  for (var i = 0; i < editables.length; i++) {
    var el = editables[i];
    if (el.tagName !== 'INPUT' && el.tagName !== 'TEXTAREA') {
      var dp = (el.getAttribute && el.getAttribute('data-placeholder')) || '';
      var cls = (el.className || '') + ' ';
      // 匹配：data-placeholder 含 "描述"/"正文"，或 className 含 textarea/container
      if (/描述|正文|作品描述/i.test(dp) ||
          cls.indexOf('textarea') !== -1 ||
          cls.indexOf('container') !== -1) {
        target = el;
        break;
      }
    }
  }
  if (!target) {
    // 兜底：第一个 contenteditable div（排除标题区域）
    for (var j = 0; j < editables.length; j++) {
      var el2 = editables[j];
      if (el2.tagName !== 'INPUT' && el2.tagName !== 'TEXTAREA') {
        target = el2;
        break;
      }
    }
  }
  if (!target) return { ok: false, reason: 'no-editable' };

  // 2. 构造完整文本（正文 + 话题）
  var fullText = '用户输入的正文内容';
  var tags = ['话题1', '话题2'];  // 从 PublishRequest.tags 获取
  if (tags && tags.length > 0) {
    var tagText = tags.slice(0, 10).map(function(t) {
      return '#' + (t.startsWith('#') ? t.slice(1) : t);
    }).join(' ');
    if (fullText) fullText = fullText + '\n' + tagText;
    else fullText = tagText;
  }

  // 3. 字数限制（最多 1000 字）
  if (fullText.length > 1000) {
    fullText = fullText.slice(0, 999) + '\u2026';
  }

  // 4. execCommand 写入（确保 React state 更新）
  target.focus();
  try { document.execCommand('selectAll'); } catch(e) {}
  try { document.execCommand('delete'); } catch(e) {}
  try {
    document.execCommand('insertText', false, fullText);
  } catch(eCmd) {
    // 兜底：innerText
    target.innerText = fullText;
    try { target.dispatchEvent(new Event('input', { bubbles: true })); } catch(eEv) {}
  }
  return { ok: true, method: 'contenteditable', textLength: fullText.length };
})()
```

### 难点 3：上传后页面跳转（导航跟踪器）

抖音创作者中心在上传后会触发 SPA 路由跳转：
- `content/upload` → `content/upload?default-tab=3` → `content/publish?itemId=xxx`

这导致 `executeJavaScript` 在跳转瞬间可能失败（frame disposed）。

**导航跟踪器的处理方式**：

```typescript
// attachNavigationTracker 内部
lastNavigationAt = Date.now();
navigateCount = 0;

// 监听所有会导致 frame 变化的事件
win.webContents.on('did-navigate-in-page',
  (_e, url) => { lastNavigationAt = Date.now(); navigateCount++; });
win.webContents.on('did-navigate',
  (_e, url) => { lastNavigationAt = Date.now(); navigateCount++; });
win.webContents.on('did-frame-navigate',
  (_e, url) => { lastNavigationAt = Date.now(); navigateCount++; });

// waitForStable — 等待至少 minStableMs 没有新导航
function waitForStable(minStableMs = 1000, timeoutMs = 10000) {
  const start = Date.now();
  while (Date.now() - start < timeoutMs) {
    if (!win.webContents.isLoading() &&
        Date.now() - lastNavigationAt >= minStableMs) {
      return;  // 稳定
    }
    await sleep(200);
  }
  log('warn', 'navigation', `稳定超时 ${timeoutMs}ms，继续`);
}
```

**关键调用位置**：
- uploadViaCDP 之后 → `tracker.waitForStable(1500, 15000)`
- 填写标题之前 → `tracker.waitForStable(800, 8000)`
- evalJS 内部已有重试机制（6 次，指数退避）

## 六、失败回退

| 失败场景 | 处理方式 | 用户体验 |
|:---|:---|:---|
| 登录态过期 | 保持窗口打开 | 用户可手动扫码登录，或在账号管理页面刷新账号 |
| 文件上传失败 | 保持窗口打开 | 用户可手动上传图片 |
| 标题/描述填写失败 | 保持窗口打开 | 用户可手动填写 |
| 发布按钮未找到 | 保持窗口打开 | 用户可手动点击发布 |
| 发布后未检测到跳转 | 保持窗口打开 | 用户可手动确认发布结果 |
| 字数超限 | 自动截断，记录 warning | 透明处理，无需用户干预 |

## 七、与视频发布的差异对比

| 维度 | 抖音视频发布 | 抖音图文发布 |
|:---|:---|:---|
| 发布 URL | `default-tab=1` 或无参数 | `default-tab=3` |
| 上传文件类型 | `.mp4` / `.mov` 视频 | `.jpg` / `.png` / `.webp` 图片 |
| 标题输入框 | input[type=text] | input[type=text] |
| 正文编辑器 | contenteditable div | contenteditable div |
| 发布按钮 | `.button-dhlUZE.primary-*` | `.button-dhlUZE.primary-*`（相同） |
| 发布流程 | 上传 → 转码 → 填写 → 发布 | 上传 → 填写 → 发布（无转码） |
| 发布耗时 | 10~60 秒（取决于视频时长） | 5~15 秒 |
