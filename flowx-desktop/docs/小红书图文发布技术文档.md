## 一、目标页面结构分析

### 1.1 发布页 URL

```
图文发布 URL: https://creator.xiaohongshu.com/publish/publish
```

该 URL 同时支持视频发布和图文发布。页面会根据上传的文件类型自动切换模式。上传图片文件后自动进入图文发布模式。

### 1.2 页面核心 DOM 结构

#### 文件上传区域

```html
<!-- 上传区域（页面中部大上传按钮区域 -->
<div class="upload-wrapper">
  <input type="file" accept="image/png, image/jpeg" multiple />
  <button class="d-button">上传图片</button>
</div>
```

**关键点**：
- 存在多个 `input[type="file"]` 元素
- `accept` 属性区分 image 和 video 模式
- uploadViaCDP 会自动扫描并选择 accept 含 `image/` 的 input

#### 标题输入框

```html
<!-- 标题输入框（普通 input，placeholder 提示文字
<input type="text" placeholder="填写标题会有更多赞哦" />
```

**特殊限制**：小红书标题最多 **20 个字符**。超过会被系统截断并提示。

实现中：
```typescript
const TITLE_LIMIT = 20;
// 发布前 truncate，记录 warning 日志
```

#### 正文编辑区（ProseMirror 富文本）

```html
<!-- 正文编辑区是 ProseMirror/Tiptap 富文本编辑器 -->
<div class="ProseMirror" contenteditable="true" role="textbox">
  <p>第一个段落</p>
  <p>第二个段落</p>
</div>
```

**ProseMirror 的特殊性**：
- ProseMirror 有内部的 `state.doc` 数据模型
- 直接设置 `innerText` / `textContent` 不会更新 state
- 下次 keydown 事件触发时，ProseMirror 会根据 state 重新渲染，内容被清空
- 必须通过 `document.execCommand('insertText')` 模拟原生输入

#### 发布按钮（Closed Shadow DOM）

```html
<xhs-publish-btn
  submit-text="发布笔记"
  save-text="暂存离开"
  submit-disabled="false">
  <!-- #shadow-root (closed)  ← 外部 JS 无法直接访问
    <div class="publish-page-publish-btn">
      <button class="ce-btn bg-red">发布笔记</button>
    </div>
  </xhs-publish-btn>
</xhs-publish-btn>
```

**为什么 JS click() 不生效**：

```javascript
// ❌ 不生效：
const btn = document.querySelector('xhs-publish-btn');
btn.click();
// 事件监听器绑定在 shadow DOM 内部的 <button> 上，
// 而不是 host 元素 <xhs-publish-btn>。

// ✅ 生效（CDP 穿透）：见第三节详细说明
```

## 二、完整发布流程

### 2.1 PlatformDispatcher 分发

```
PublishEngine.submit(req)
  └─> PlatformDispatcher.createExecutor('xiaohongshu', 'image')
        └─> xiaohongshu.publishImage(accountId, req, onProgress)
              └─> 完全独立的图文发布流程（与 publishVideo 互不影响）
```

### 2.2 发布流程图

```
1. 窗口初始化
   └─> makePublishWindow(accountId)
         · 创建 BrowserWindow（sandbox + contextIsolation + partition: persist:account_${id}
         · 监听 render-process-gone → 自动 reload
   └─> attachNavigationTracker(win, log)
         · 监听 did-navigate / did-navigate-in-page / did-frame-navigate
         · 记录 lastNavigationAt
         · 提供 isNavigating() / waitForStable()

2. 导航到发布页
   └─> win.loadURL('https://creator.xiaohongshu.com/publish/publish')
   └─> tracker.waitForStable(1500, 15000)  ← 等待 1.5s 静默期

3. 登录态检测
   └─> evalJS(win, buildDetectLoggedInScript(LOGIN_KEYWORDS), '检测登录')
         · 检查 URL 不包含 login/signup
         · 检查 body.innerText 命中 "创作中心"/"发布笔记" 等关键词
         · 检查存在 account-name / nickname 等账号相关元素

4. 文件上传（CDP）
   └─> uploadViaCDP(win, files, log, contentType='image')
         · CDP DOM.getDocument({ depth: -1 }) 深度遍历
         · 按 accept 属性评分：含 image/ 的 +10 分
         · 跳过 aria-hidden="true" 内的 input
         · 选择评分最高的 input，DOM.setFileInputFiles(nodeId, files)
         · 手动派发 change + input 事件
   └─> sleep(1500) 等待上传处理完成
   └─> waitForUploadComplete(win, log) 轮询检测

5. 填写标题
   └─> evalJS(win, buildFillTitleScript(title), '填写标题')
         · placeholder 含 "标题" 的 input
         · execCommand('selectAll') → delete → insertText
         · 超过 20 字自动 truncate

6. 填写正文（ProseMirror）
   └─> evalJS(win, buildFillContentScript(content, tags), '填写正文')
         · 查找 contenteditable 元素
         · ProseMirror 逐行 insertText：
         · 每行执行一次 insertText(false, line + '\n')
         · 话题（tags）：转换为 #话题1 #话题2 行文本
         · 超过 1000 字自动 truncate

7. 点击发布按钮（关键：三层策略）
   └─> evalJS(win, buildClickPublishScript(), '点击发布') ← 第一层：JS click()
   └─> cdpClickPublishButton(win, log, ['发布笔记', '发布'])  ← 第二层+第三层：CDP 穿透
   └─> sleep(1200) 等待点击生效

8. 发布结果检测
   └─> evalJS(win, '检查 URL 是否跳转到 /home 或显示发布成功')
         · 成功：跳转到创作者中心首页，或显示 "发布成功" 提示
         · 失败：停留在发布页，或显示错误信息
   └─> sleep(3000) 等待跳转完成
   └─> evalJS(win, `document.body.innerText.includes('发布成功') || !location.href.includes('/publish')`)

9. 完成/清理
   └─> 成功：win.destroy() 自动关闭
   └─> 失败：保持窗口打开，让用户查看错误信息或手动发布
```

## 三、核心难点与解决方案

### 难点 1：ProseMirror 富文本填写

**问题**：正文编辑区是 ProseMirror/Tiptap 富文本编辑器。

```javascript
// ❌ 失败方案 1：直接设置 innerText
editable.innerText = content;
// ProseMirror 内部 state 未更新，下次 keydown 还原内容

// ❌ 失败方案 2：直接设置 value
editable.value = content;  // contenteditable 元素没有 value 属性

// ✅ 正确方案：execCommand('insertText') 模拟原生输入
// 在 buildFillContentScript 内实现
function buildFillContentScript(content: string, tags: string[]): string {
  const contentJson = JSON.stringify(content);
  const tagsJson = JSON.stringify(tags);
  return `(function() {
    // 1. 查找 contenteditable 元素
    var editables = document.querySelectorAll('[contenteditable="true"]');
    var target = null;
    for (var i = 0; i < editables.length; i++) {
      var el = editables[i];
      // 跳过输入框选择器（标题区域）
      if (el.tagName !== 'INPUT' && el.tagName !== 'TEXTAREA') {
        var cls = (el.className || '') + ' ';
        if (cls.indexOf('ProseMirror') !== -1 ||
            cls.indexOf('note-editor') !== -1 ||
            cls.indexOf('editor') !== -1) {
          target = el;
          break;
        }
      }
    }
    if (!target) {
      // 兜底：取第一个 contenteditable
      if (editables.length > 0) target = editables[0];
    }
    if (!target) return { ok: false, reason: 'no-editable' };

    // 2. 构造完整文本（正文 + 话题）
    var fullText = ${contentJson};
    var tags = ${tagsJson};
    if (tags && tags.length > 0) {
      var tagText = tags.map(function(t) {
        return '#' + (t.startsWith('#') ? t.slice(1) : t);
      }).join(' ');
      if (fullText) fullText = fullText + '\\n' + tagText;
      else fullText = tagText;
    }

    // 3. 字数限制（最多 1000 字）
    if (fullText.length > 1000) {
      fullText = fullText.slice(0, 999) + '\\u2026';
    }

    // 4. ProseMirror 逐行 insertText
    target.focus();
    try { document.execCommand('selectAll'); } catch(e) {}
    try { document.execCommand('delete'); } catch(e) {}

    var lines = fullText.split('\\n');
    for (var j = 0; j < lines.length; j++) {
      if (lines[j]) {
        try {
          document.execCommand('insertText', false, lines[j]);
        } catch(eCmd) {}
      }
      if (j < lines.length - 1) {
        try {
          document.execCommand('insertText', false, '\\n');
        } catch(eNl) {}
      }
    }
    return { ok: true, method: 'prosemirror', textLength: fullText.length };
  })()`;
}
```

**关键洞察**：ProseMirror 监听浏览器原生 `input` 事件来更新内部 `state.doc`。`execCommand('insertText')` 会触发原生 `input` 事件，因此能正确更新 state。

### 难点 2：Closed Shadow DOM 的发布按钮

**问题**：`<xhs-publish-btn>` 是自定义 Web Component，内部 shadow DOM 对外封闭。

```javascript
// ❌ 失败方案 1：直接点击 host 元素
const btn = document.querySelector('xhs-publish-btn');
btn.click();
// 事件监听器绑定在 shadow DOM 内部的 <button>，不是 host

// ❌ 失败方案 2：查询 shadow DOM 内部元素
const inner = document.querySelector('xhs-publish-btn button');
// null — CSS 选择器无法穿透 closed shadow DOM

// ✅ 正确方案：Chrome DevTools Protocol 穿透
// 由 shared.ts 的 cdpClickPublishButton 实现
```

**CDP 穿透实现原理**：

```typescript
// cdpClickPublishButton(win, log, keywords)
//
// 1. CDP 连接到窗口的 webContents
//    ↓
// 2. DOM.getDocument({ depth: -1, pierce: true })
//    ← 关键参数 pierce: true — 强制穿透 shadow DOM
//    ↓
// 3. 遍历返回的节点树
//    · 收集所有 <button> 元素（含 shadow DOM 内的）
//    · 收集含关键字文本的元素
//    · 收集 a / div[role=button] / span[role=button]
//    ↓
// 4. 对每个候选 nodeId：
//    DOM.getBoxModel(nodeId)
//    ← content[0..3] = [x1, y1, x2, y2]
//    ↓
// 5. 计算中心坐标 (centerX, centerY)
//    ↓
// 6. Input.dispatchMouseEvent 合成鼠标事件
//    · type: 'mouseMoved'    → 模拟鼠标移动到按钮
//    · type: 'mousePressed'  → 按下左键
//    · type: 'mouseReleased' → 松开左键
//    · button: 'left', clickCount: 1
//    ↓
// 7. 合成鼠标事件不受 shadow DOM 隔离
//    ← OS 级别的鼠标事件，直接派发给页面
```

**三层策略的具体代码**（在 xiaohongshu.ts publishImage 内）：

```typescript
// 第一层：尝试 JS click()
onProgress(85, '点击发布按钮…');
let clicked = false;
const clickResult: any = await evalJS(
  win, buildClickPublishScript(), '点击发布', log,
);
if (clickResult && clickResult.clicked) clicked = true;

// 第二层 + 第三层：CDP 穿透点击（关键！）
try {
  const cdpOk = await cdpClickPublishButton(win, log, [
    '发布笔记', '发布', '立即发布',
  ]);
  if (cdpOk) {
    log('info', 'click-publish', 'CDP 穿透式点击成功');
    clicked = true;
  }
} catch (cdpErr) {
  log('warn', 'click-publish',
    `CDP 点击异常: ${cdpErr instanceof Error ? cdpErr.message : String(cdpErr)}`);
}

if (!clicked) {
  log('warn', 'click-publish', '所有点击方式均未命中发布按钮');
  return makeFailedResult(accountId, 'xiaohongshu',
    '未找到发布按钮或点击失败', startedAt);
}
await sleep(1200);
```

### 难点 3：React 受控组件标题被清空

**问题**：小红书标题输入框是 React 受控组件。

```javascript
// React 代码（概念）：
state = { title: '' };

<input
  type="text"
  value={this.state.title}         ← 渲染值与 state 同步
  onChange={e => this.setState({
    title: e.target.value,        ← 只能通过 onChange 更新 state
  })}
/>
```

时序问题：
```
t1: JS 设置 input.value = '我的标题'  ← 改了 DOM，没改 state
t2: React 重渲染（其他字段更新时触发  ← 重新渲染 input，state 还是 ''
t3: input.value = ''                   ← 标题被清空！
```

**解决方案**：用 `execCommand('insertText')` 代替直接赋值：

```javascript
// ✅ buildFillTitleScript 内部实现
input.focus();
try { document.execCommand('selectAll'); } catch(e) {}
try { document.execCommand('delete'); } catch(e) {}
document.execCommand('insertText', false, text);
// → execCommand 触发原生 input 事件
// → React 监听该事件，调用 onChange(e)
// → setState({ title: text })
// → 下次重渲染保持正确值
```

## 四、字数与标签限制

### 4.1 标题限制（最多 20 字）

```typescript
const TITLE_LIMIT = 20;

function buildFillTitleScript(title: string): string {
  // 超过 20 字自动截断
  let displayTitle = title;
  if (displayTitle.length > TITLE_LIMIT) {
    displayTitle = displayTitle.slice(0, TITLE_LIMIT - 1).trimEnd() + '\u2026';
  }
  // 记录 warning 日志
  // log('warn', 'fill-title',
  //   `标题超过 ${TITLE_LIMIT} 字，已从 ${title.length} 截断`)
  const titleJson = JSON.stringify(displayTitle);
  return `(function() {
    // ... find input with placeholder 含 "标题"
    input.focus();
    try { document.execCommand('selectAll'); } catch(e) {}
    try { document.execCommand('delete'); } catch(e) {}
    document.execCommand('insertText', false, ${titleJson});
    return { ok: true, method: 'input-execcommand' };
  })()`;
}
```

### 4.2 正文限制（最多 1000 字）

正文限制在 `buildFillContentScript` 内实现，见第三节「ProseMirror 富文本填写」。

### 4.3 话题限制（最多 10 个）

话题在 `buildFillContentScript` 内处理，将用户输入的 tags 转换为 `#话题1 #话题2` 行文本。超过 10 个时截断前 10 个。

## 六、失败回退

| 失败场景 | 处理方式 | 用户体验 |
|:---|:---|:---|
| 登录态过期 | 保持窗口打开 | 用户可手动扫码登录，或在账号管理页面刷新账号 |
| 文件上传失败 | 保持窗口打开 | 用户可手动上传图片 |
| 发布按钮未找到 | 保持窗口打开 | 用户可手动点击发布 |
| 发布后未检测到跳转 | 保持窗口打开 | 用户可手动确认发布结果 |
| 字数超限 | 自动截断，记录 warning | 透明处理，无需用户干预 |
