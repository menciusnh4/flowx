# 小红书文章（长文）发布技术文档

> 发布入口：`https://creator.xiaohongshu.com/publish/publish?source=official&from=menu&target=article`

## 一、文章发布 vs 图文发布

| 维度 | 图文发布（image） | 文章发布（article） |
|------|------------------|-------------------|
| 入口 URL | 同默认发布页，需检测 tab 切换 | `?target=article` 专用参数进入文章管理页 |
| 编辑模式 | 填写标题 + 正文 + 上传封面 → 点击"发布笔记" | **点击"新的创作" → 编辑正文 → 一键排版 → 下一步 → 发布**（多步流程） |
| 标题限制 | 20 字 | **64 字** |
| 正文限制 | 1000 字 | **无限制** |
| 话题标签 | 最多 10 个 | 可选 |
| 封面图 | 必填 | 可选（在排版阶段自动生成或上传） |
| 发布按钮 | `xhs-publish-btn`（shadow DOM） | `button.ce-btn.bg-red`（shadow DOM web component） |

## 二、核心流程

```
┌─────────────────────────────────────────────────────────────────┐
│  1) loadURL(https://creator.xiaohongshu.com/publish/publish?   │
│          source=official&from=menu&target=article)              │
│                          ↓                                      │
│  2) 检测是否已在编辑器中（页面状态检测脚本）                     │
│     - hasTitleInput: 检测是否存在 <textarea> 或类似标题输入     │
│     - hasEditor: 检测是否存在 contenteditable 的 ProseMirror   │
│     - createCount: 检测页面中"新的创作"按钮的可见性             │
│                          ↓                                      │
│  3) 不在编辑器？ → 点击"新的创作"按钮进入编辑器                 │
│     (Class 选择器优先: button.bg-red / button.custom-button)   │
│                          ↓                                      │
│  4) 填写标题（64 字限制）                                       │
│     buildFillTitle(title, ARTICLE_TITLE_LIMIT=64)              │
│     - 定位 <textarea>（非图文编辑器）                           │
│     - execCommand('insertText') 逐字模拟用户输入                 │
│                          ↓                                      │
│  5) 填写正文（无字数限制，ProseMirror 富文本编辑器）             │
│     buildFillContent(content, null)                            │
│     - 自动检测 ProseMirror 编辑器结构（.ProseMirror / editor） │
│     - execCommand('insertText') 模拟用户输入                    │
│     - 自动换行与段落分割                                         │
│                          ↓                                      │
│  6) 🔑 多步点击发布流程（3 步）                                  │
│     ├─ 第 1 步：点击"一键排版"                                   │
│     │   - 按钮: button.ce-btn.bg-red（红色按钮）                │
│     │   - 等待排版完成（轮询 button 文本变化）                   │
│     │   - 按钮文本从"一键排版"→"下一步"（同一按钮动态变化）       │
│     │   - 轮询策略: 每秒读取一次按钮文本，直到出现"下一步"       │
│     │   - 超时: 10 秒                                           │
│     │                                                          │
│     ├─ 第 2 步：点击"下一步"                                    │
│     │   - 同一按钮: button.ce-btn.bg-red                       │
│     │   - 等待页面导航稳定（waitForStable 5 秒）                │
│     │   - URL 从 publish?target=article → publish? 继续         │
│     │   - 轮询策略: 每秒检测按钮文本是否变化                    │
│     │                                                          │
│     └─ 第 3 步：点击"发布"                                      │
│         - 按钮在 closed shadow DOM 中: <xhs-publish-btn>       │
│         - 🔑 必须通过 CDP 深度扫描（pierce:true）穿透 shadow DOM │
│         - 扫描策略:                                             │
│             1) JS querySelector 先尝试（10 次轮询）              │
│             2) 全失败 → 降级为 CDP 穿透式点击                    │
│             3) CDP 扫描所有候选节点（通常 30+）                  │
│             4) 按 score = (keyword-hit × 2000) + (size/10) 排序 │
│             5) 从高到低依次点击，每次点击后检测 URL 变化         │
│             6) URL 变化（离开 publish 页或出现 success） → 成功  │
│                          ↓                                      │
│  7) 等待发布结果（60 秒超时）                                    │
│     探测 URL: /publish/success 或 published=true                │
│                          ↓                                      │
│  8) 结束：PublishItemProgress{success:true, resultUrl: ...}   │
└─────────────────────────────────────────────────────────────────┘
```

## 三、关键技术点

### 3.1 文章专用 URL 的登录态检测

`?target=article` URL 打开后，登录态检测与图文发布相同：
- 读取 `document.title` 是否包含"小红书创作服务平台"
- 检查 URL 是否包含 `/publish/`
- 扫描 DOM 文本中包含"发布笔记"/"新的创作"等关键词

### 3.2 页面状态检测：判断"是否在编辑器内"

```javascript
// 关键检测脚本
{
  hasTitleInput:  document.querySelector('textarea') != null
                || document.querySelector('[contenteditable="true"][role="textbox"]') != null
  hasEditor:      document.querySelector('[contenteditable="true"]') != null
  createCount:    number of buttons/div containing "新的创作"
}
```

- 如果页面上既有内容可编辑区域又有"新的创作"按钮 → **已经在编辑器中**，跳过点击入口
- 如果只有"新的创作"按钮而没有可编辑区域 → **在文章管理列表页**，需点击进入

### 3.3 进入编辑器：点击"新的创作"按钮

```typescript
// 优先级：
//   1) Class 选择器命中（button.bg-red / button.custom-button / button.ce-btn）
//   2) 文本精确匹配("新的创作") <button> / <a> / [role="button"]
//   3) 全文本搜索 → 回溯父元素 → 找到可点击元素
// 点击后等待：waitForStable(1500ms) + sleep(2000ms)
```

### 3.4 标题与正文填写

**标题填写**：`buildFillTitle(title, 64)` — 与图文发布共享同一套函数
- 优先定位 `<textarea>`（文章编辑器的标题字段）
- 使用 `document.execCommand('insertText')` 逐字插入，触发受控组件更新
- 超过 64 字时自动截断并记录警告

**正文填写**：`buildFillContent(content, null)` — null 表示无字数限制
- ProseMirror 编辑器自动检测：`.ProseMirror` 类名 / `p` 子元素 / `doc-editor`
- 内容插入方式：`execCommand('insertText')`，确保触发编辑器的 insert 事务
- 每 1000 字左右自然分段，避免一次性插入过长文本

### 3.5 🔑 多步发布流程（文章 vs 图文的最大差异）

**图文发布**：单步点击"发布笔记"即可

**文章发布**：点击"发布笔记"按钮只会跳回图文发布页！必须按以下顺序操作：

```
Step 1: 点击"一键排版"  (button.ce-btn.bg-red)
         ↓ wait 5~10 秒，等待排版完成
         按钮文本变化为"下一步"

Step 2: 点击"下一步"    (同一 button.ce-btn.bg-red)
         ↓ wait 3~5 秒，导航到发布配置页
         按钮文本变化为"发布"

Step 3: 点击"发布"      (button.ce-btn.bg-red)
         ↓ wait 2~3 秒
         URL 跳转到 /publish/success → ✅ 成功
```

### 3.6 轮询等待策略

两步之间的"等待"不是固定 sleep，而是**轮询检测**：

```typescript
// 每 1 秒读取按钮文本
//  如果文本仍为上一步的内容（如"一键排版"）→ 继续等待
//  如果文本变为下一步关键词（如"下一步"） → 继续执行
//  如果 10 秒内无变化 → 视为失败
//
// 如果 JS querySelector 无法找到按钮 → 降级为 CDP 穿透 shadow DOM 读取
```

### 3.7 发布按钮在 Closed Shadow DOM 中

文章最后一步的"发布"按钮与图文发布的"发布笔记"按钮结构相同：

```
<xhs-publish-btn>   <!-- closed shadow DOM host element -->
  └─ #shadow-root (closed)
      └─ <button class="ce-btn bg-red">发布</button>
```

**穿透方式**：通过 Chrome DevTools Protocol 深度扫描 DOM（`pierce:true`），然后用 `Input.dispatchMouseEvent` 合成鼠标点击事件。详见 `小红书图文发布技术文档.md` 第 4 节。

### 3.8 发布结果检测

文章发布成功后，URL 跳转到：
- `https://creator.xiaohongshu.com/publish/success?source=official...`
- 或返回 `https://creator.xiaohongshu.com/publish/publish?source=official&published=true`

`buildPublishResultProbeScript()` 新增专门检测：
- `isArticleSuccess = /publish/success 或 published=true`
- 命中即视为 `success=true`，不再依赖 body 文本关键词

## 四、已知限制

| 限制 | 说明 |
|------|------|
| 多步点击依赖页面 JS 渲染完成 | 排版过程需要 3-10 秒，期间按钮不可点击 |
| 按钮文本是中文 | "一键排版/下一步/发布"，与平台语言设置绑定 |
| Shadow DOM 结构随版本变化 | `xhs-publish-btn` 的内部结构可能更新 |
| 文章编辑器字段结构不稳定 | 标题字段可能是 `<textarea>` 或 contenteditable `<div>` |

## 五、与图文发布的代码隔离

文章发布 (`publishArticle`) 与图文发布 (`publishImage`) **共用**：

| 共享模块 | 说明 |
|---------|------|
| `buildFillTitle()` | 标题填写函数（不同字数限制由参数控制） |
| `buildFillContent()` | 正文内容填充（支持 ProseMirror） |
| `cdpClickPublishButton()` | CDP 穿透 shadow DOM 点击 |
| `waitForStable` / `attachNavigationTracker` | 页面导航稳定检测 |

文章发布 **独立实现**：

| 独立实现 | 说明 |
|---------|------|
| `PUBLISH_ARTICLE_URL` 常量 | 文章发布专用入口 URL |
| `ARTICLE_TITLE_LIMIT = 64` | 标题字数限制（独立于图文的 20 字） |
| 文章编辑器入口检测逻辑 | 判断当前页是否为文章管理列表，需点击"新的创作" |
| 多步点击流程 | 一键排版 → 下一步 → 发布，完全独立于图文的单步发布 |
| 文章发布结果检测 | 识别 `/publish/success` 和 `published=true` URL |

> 架构上：修改文章发布逻辑不需要触碰 `publishImage` 或 `publishVideo` 代码路径。
