> 版本: v1.1（基于 PlatformDispatcher + 工厂方法模式
> 适用范围: Windows 10/11，FlowX 桌面客户端 v1.1+
> 归档时间: 2025-06
> 对应源码: `src/main/services/platforms/kuaishou.ts` → `publishImage()`

## 一、目标页面结构分析

### 1.1 发布页 URL（图文模式）

```
图文发布 URL: https://cp.kuaishou.com/article/publish/video?tabType=2
                                                                 ↑
                                                           tabType=2 = 图文模式
                                                           tabType=1 = 视频模式（默认）
```

**为什么用 tabType=2**：
- 快手创作者中心的发布页是一个单页面，通过 `tabType` 参数切换发布模式
- `tabType=2` 直接进入图文发布模式，省去点击切换 tab 的步骤
- 这是图文发布的专用入口

### 1.2 页面核心 DOM 结构

快手使用 Element UI（Vue.js 组件库），DOM 结构有明显的 class 命名规律。

#### 文件上传区域

```html
<!-- 图文上传区域（tabType=2 时自动激活） -->
<div class="upload-container" data-v-xxxxx>
  <input type="file" accept="image/*" multiple style="display: none;" />
  <div class="_upload-btn" data-v-xxxxx>
    <span>上传图片</span>
  </div>
</div>
```

**关键点**：
- `uploadViaCDP(win, files, log, contentType='image')` 会自动选择 accept 含 `image/` 的 input
- 支持多图片上传
- 上传成功后页面保持在同一 URL，不触发 SPA 路由跳转（与抖音不同）

#### 标题输入框 — **图文模式无独立标题！**

**重要发现**：快手图文发布与视频发布的重要差异：

| 模式 | 标题字段 | 描述字段 |
|:---|:---|:---|
| 视频发布 | ✅ contenteditable div（placeholder="作品标题"） | ✅ contenteditable div（placeholder="添加合适的话题和描述"） |
| 图文发布 | ❌ **无独立标题字段** | ✅ contenteditable div（描述 = 图文的核心文本） |

**因此**，快手图文发布流程中：
- 不调用 `buildFillTitleScript`（没有标题输入框）
- 用户输入的 title 会被合并到描述文本开头
- 这与抖音/小红书的图文发布都不同

#### 描述输入框（图文的核心文本区域）

```html
<!-- 描述/内容输入框（contenteditable div） -->
<div contenteditable="true"
     class="_input _textarea"
     data-v-xxxxx
     placeholder="添加合适的话题和描述">
  <!-- 用户输入的文本内容 -->
</div>
```

快手描述输入框是 contenteditable div，与抖音描述区域类似。选择器匹配 `_input` 和 `_textarea` 类名。

#### 发布按钮

```html
<!-- 发布按钮（div + button 组合，Element UI 风格） -->
<div class="_button_3a3lq_1 _button-primary_3a3lq_60" role="button">
  <span>发布</span>
</div>
```

**关键选择器**：
- `._button_3a3lq_1._button-primary_3a3lq_60` — 最精确匹配
- `div[role="button"]` 含文本"发布" — 兜底选择器
- `span:contains('发布')` 的父元素 — 兜底

发布按钮是标准 DOM div（role=button），**不在 shadow DOM 内**，标准 JS click() 即可生效。

## 二、完整发布流程

### 2.1 PlatformDispatcher 分发

```
PublishEngine.submit(req)
  └─> PlatformDispatcher.createExecutor('kuaishou', 'image')
        └─> kuaishou.publishImage(accountId, req, onProgress)
              └─> 完全独立的图文发布流程（与 publishVideo 互不影响）
```

### 2.2 发布流程图

```
1. 窗口初始化
   └─> makePublishWindow(accountId)
         · BrowserWindow（sandbox + contextIsolation + partition）
         · 监听 render-process-gone
   └─> attachNavigationTracker(win, log)
         · 导航跟踪器（处理 URL 参数变化）

2. 导航到图文发布页（关键：tabType=2）
   └─> win.loadURL('https://cp.kuaishou.com/article/publish/video?tabType=2')
   └─> tracker.waitForStable(1500, 15000)  ← 1.5s 静默期

3. 登录态检测
   └─> evalJS(win, buildDetectLoggedInScript(), '检测登录态')
         · 检查 URL 不包含 login/signin/redirect
         · 检查 body.innerText 命中 "发布作品"/"作品管理" 等关键词
         · 检查存在 ".header-info-card .user-name" 等账号元素

4. 文件上传（CDP）
   └─> uploadViaCDP(win, files, log, contentType='image')
         · CDP DOM.getDocument({ depth: -1 }) 深度遍历
         · 按 accept 属性评分排序（含 image/ 的优先）
         · DOM.setFileInputFiles(nodeId, files) 注入文件
         · 派发 change + input 事件
   └─> sleep(1500) 等待上传完成
   └─> waitForUploadComplete(win, log) 轮询检测

5. 填写描述 + 话题（**注意：图文无独立标题！**）
   └─> evalJS(win, buildFillContentScript(title + '\n' + content, tags), '填写描述')
         · contenteditable div（placeholder="添加合适的话题和描述"）
         · title 合并到描述文本开头（快手图文无独立标题字段）
         · 话题：转换为 #话题1 #话题2 行文本
         · 最多 500 字，超过自动 truncate
         · 最多 4 个标签（⚠️ 快手限制：超过 4 个会导致发布失败）

6. 点击发布按钮
   └─> evalJS(win, buildClickPublishScript(), '点击发布')
         · 查找 div._button-primary_3a3lq_60 或含"发布"文本的元素
         · element.click() — 标准 DOM click
   └─> sleep(1500) 等待点击生效

7. 发布结果检测
   └─> sleep(3000) 等待页面跳转或反馈
   └─> evalJS(win, verifyScript, '验证发布结果')
         · 成功：URL 跳转到 /article/manage，或 body 含"发布成功"
         · 失败：停留在 /publish，或 body 含错误信息（如"话题数量超过限制"）

8. 完成/清理
   └─> 成功：win.destroy() 自动关闭
   └─> 失败：保持窗口打开，让用户查看/手动发布
```

## 三、核心难点与解决方案

### 难点 1：图文无独立标题字段

**问题**：快手图文发布页面没有独立的「标题」输入框。视频发布时有标题字段，但图文发布时标题字段不存在或被隐藏。

**错误流程（会失败）**：
```
尝试填写标题 → 找不到标题 input → evalJS 失败 → 整个发布流程中止
```

**正确流程**：
```
1. 检测当前是否为图文模式（URL 含 tabType=2，或页面含"图文"等提示文本）
2. 如果是图文模式：**跳过标题填写**，将 title 合并到描述文本开头
3. 直接填写描述字段：
   contenteditable div（placeholder="添加合适的话题和描述"）
   写入格式：
   [title]           ← 用户输入的标题放在第一行
   [content]         ← 用户输入的正文内容
   #tag1 #tag2       ← 话题（最多 4 个）
```

**代码实现方式**（kuaishou.ts publishImage 内）：
```typescript
// 在 publishImage 内
const combinedContent = [
  req.title,            // 标题放在描述开头
  req.content,          // 正文
].filter(Boolean).join('\n');
// 然后调用 buildFillContentScript(combinedContent, req.tags?.slice(0, TAG_MAX))
```

### 难点 2：标签最多 4 个

**问题**：快手图文发布严格限制标签数量。超过 4 个会触发「参数不合法」错误，导致发布失败。

**失败场景**：
```
用户输入 tags = ['日常', '美食', '生活', '分享', '记录']  ← 5 个标签
→ buildFillContentScript 全部写入
→ 点击发布
→ 页面提示「参数不合法」
→ 发布失败
```

**解决方案**：
```typescript
const TAG_MAX = 4;

// 在 publishImage 内
const limitedTags = (req.tags || []).slice(0, TAG_MAX);
if (req.tags && req.tags.length > TAG_MAX) {
  log('warn', 'fill-tags',
    `快手限制最多 ${TAG_MAX} 个标签，已从 ${req.tags.length} 个截断为 ${TAG_MAX} 个`);
}
// 然后用 limitedTags 填充描述区域
```

**为什么这是图文特有问题**：快手视频发布也有类似限制，但图文发布限制更严格（4 个）。视频发布的限制在 6~10 个之间（版本差异）。

### 难点 3：contenteditable 描述填写

快手描述区域是普通 contenteditable div（Element UI 的 v-textarea 组件）。

**填写脚本的完整逻辑**：

```javascript
(function() {
  // 1. 查找描述输入框
  //    优先级：
  //      a) contenteditable div 含 placeholder="添加合适的话题和描述"
  //      b) contenteditable div 含 class "_textarea" / "_input"
  //      c) 第一个 contenteditable div（排除标题区域）
  var editables = document.querySelectorAll('[contenteditable="true"]');
  var target = null;

  for (var i = 0; i < editables.length; i++) {
    var el = editables[i];
    // 跳过 INPUT / TEXTAREA（这些不是 contenteditable div）
    if (el.tagName === 'INPUT' || el.tagName === 'TEXTAREA') continue;

    var ph = (el.getAttribute && el.getAttribute('placeholder')) || '';
    var cls = (el.className || '') + ' ';

    // 匹配：placeholder 含 "描述"/"话题"/"内容"
    // 或 className 含 "_textarea"/"_input"
    if (/描述|话题|作品描述|添加合适的/i.test(ph) ||
        cls.indexOf('_textarea') !== -1 ||
        cls.indexOf('_input') !== -1) {
      target = el;
      break;
    }
  }

  if (!target) {
    // 兜底：第一个 contenteditable div
    for (var j = 0; j < editables.length; j++) {
      var el2 = editables[j];
      if (el2.tagName !== 'INPUT' && el2.tagName !== 'TEXTAREA') {
        target = el2;
        break;
      }
    }
  }

  if (!target) return { ok: false, reason: 'no-editable' };

  // 2. 构造完整文本（标题 + 正文 + 话题）
  //    格式：
  //      [title]
  //      [content]
  //      #tag1 #tag2 #tag3 #tag4
  var title = "用户输入的标题";           // 从 PublishRequest.title 获取
  var content = "用户输入的正文内容";      // 从 PublishRequest.content 获取
  var tags = ['tag1', 'tag2', 'tag3', 'tag4'];  // 从 PublishRequest.tags 获取

  var parts = [];
  if (title && title.trim()) parts.push(title.trim());
  if (content && content.trim()) parts.push(content.trim());

  var combinedText = parts.join('\n');

  // 添加话题（最多 4 个，⚠️ 快手限制）
  if (tags && tags.length > 0) {
    var limitedTags = tags.slice(0, 4);  // 截断到最多 4 个
    var tagText = limitedTags.map(function(t) {
      return '#' + (t.startsWith('#') ? t.slice(1) : t);
    }).join(' ');
    if (combinedText) combinedText = combinedText + '\n' + tagText;
    else combinedText = tagText;
  }

  // 3. 字数限制（最多 500 字，⚠️ 快手限制）
  if (combinedText.length > 500) {
    combinedText = combinedText.slice(0, 499) + '\u2026';
  }

  // 4. 写入（execCommand 确保 Element UI v-model 更新）
  target.focus();
  try { document.execCommand('selectAll'); } catch(e) {}
  try { document.execCommand('delete'); } catch(e) {}
  try {
    document.execCommand('insertText', false, combinedText);
  } catch(eCmd) {
    // 兜底：innerText + input 事件
    target.innerText = combinedText;
    try { target.dispatchEvent(new Event('input', { bubbles: true })); } catch(eEv) {}
  }
  return { ok: true, method: 'contenteditable', textLength: combinedText.length };
})()
```

### 难点 4：Element UI 组件的 v-model 更新

快手使用 Vue.js + Element UI。描述字段绑定了 `v-model`（Vue 的双向绑定指令）。

**v-model 的工作原理**：

```html
<!-- Element UI 的 el-input 组件（contenteditable 版本） -->
<el-input
  v-model="description"          <!-- 双向绑定到 this.description -->
  type="textarea"                <!-- 渲染为 contenteditable div -->
  placeholder="添加合适的话题和描述"
/>
```

```javascript
// Vue 内部会监听 contenteditable 的 input 事件：
editable.addEventListener('input', (e) => {
  this.description = e.target.innerText;  // ← 更新 state
});
```

**为什么需要 execCommand**：
- 直接设置 `innerText` 不会触发 `input` 事件
- Vue 的 `v-model` 依赖 `input` 事件来更新 state
- state 不更新 → 提交时发送的是旧值（空字符串）
- `execCommand('insertText')` 会触发原生 `input` 事件
- Vue 监听到 input 事件 → 更新 `this.description` → 提交时发送正确值

## 四、字数与标签限制（严格执行）

### 4.1 描述字数限制（最多 500 字）

```
超过 500 字时：
  - 自动截断到 499 字
  - 末尾添加省略号 "\u2026"
  - 记录 warning 级别的日志
  - 不抛出异常，流程继续
```

### 4.2 标签数量限制（最多 4 个）

```
用户输入 > 4 个标签时：
  - 只保留前 4 个
  - 记录 warning 级别日志："快手限制最多 4 个标签，已从 N 个截断为 4 个"
  - 不抛出异常，流程继续
```

**为什么必须截断**：
- 快手后端严格校验参数
- 超过 4 个标签会触发「参数不合法」错误
- 返回空结果（无错误提示文本）
- 用户体验极差：窗口保持打开但无任何反馈

## 七、与视频发布的差异对比

| 维度 | 快手视频发布 | 快手图文发布 |
|:---|:---|:---|
| 发布 URL | `?tabType=1` 或无参数 | `?tabType=2` |
| 上传文件类型 | `.mp4` 等视频 | `.jpg` / `.png` 等图片 |
| 标题输入框 | ✅ contenteditable div（placeholder="作品标题"） | ❌ **无独立标题** |
| 描述输入框 | ✅ contenteditable div（"添加合适的话题和描述"） | ✅ 相同（但承担标题功能） |
| 发布按钮 | `div._button-primary_3a3lq_60` | `div._button-primary_3a3lq_60`（相同） |
| 描述字数限制 | 最多 500 字 | 最多 500 字（相同） |
| 标签数量限制 | 最多 4~6 个（版本差异） | 最多 **4 个**（严格限制） |
| 发布后跳转 | `/article/manage/video` | `/article/manage`（与视频相同） |

## 九、常见问题排查

### Q1: 发布失败，提示「参数不合法」

**原因**：标签数量超过 4 个，或描述超过 500 字

**排查**：
1. 查看日志中 `[fill-tags]` 条目，确认是否有截断警告
2. 查看日志中 `[填写描述]` 条目的 `textLength` 字段
3. 在 DevTools Console 执行：
   ```javascript
   document.querySelector('[contenteditable="true"]').innerText.length
   ```
   确认实际写入的字数

### Q2: 页面停留在上传中，无法进入编辑

**原因**：uploadViaCDP 未选择正确的 file input，或 change 事件未触发

**排查**：
1. 确认上传图片是 `.jpg` / `.png` 格式（不是其他格式）
2. 查看日志中 `[upload-cdp]` 条目，确认注入的是 accept 含 `image/` 的 input
3. 在 DevTools Console 执行：
   ```javascript
   document.querySelectorAll('input[type="file"]')
   ```
   查看每个 input 的 accept 属性

### Q3: 描述填写后被清空

**原因**：Element UI 的 v-model 未检测到内容变化

**排查**：
1. 检查浏览器 console 是否有 JS 错误
2. 在 Console 执行：
   ```javascript
   var el = document.querySelector('[contenteditable="true"]');
   el.dispatchEvent(new Event('input', { bubbles: true }));
   ```
   观察是否恢复正确值（如果恢复，说明是事件触发问题）
