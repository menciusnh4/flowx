# PRD — 全局任务选项卡 & 账号创作中心多开页签

> 文档状态：v3（原型交付稿 · 5 项决策已确认）｜ 优先级：P1（建议在合规 P0 稳定后推进）
> 作者：产品通 ｜ 依赖：合规 P0 已落地；浏览器环境隔离能力（BrowserEnvService）已具备
> 设计基线：对齐 `flowx-ui-redesign` 视觉规范（`src/renderer/assets/styles.css` 设计 token）

## 1. 背景与问题（Why）

**现状**：
- 主界面 = 左侧竖栏导航（仪表盘 / 账号管理 / 工作台 / 一键发布 / 系统配置）+ 右侧 `<router-view>` 内容区。点击左侧项即通过路由切换页面。
- 点击账号「进入创作中心」会弹出一个**独立 `BrowserWindow`**（`CreatorTabWindow`），内部已实现 Chrome 风格 tab + 工具栏 + 每 tab 一个 `WebContentsView`，且已通过 `persist:account_${id}` 分区 + `BrowserEnvService` 做了 Cookie 隔离 / 代理 IP / UA 伪装。

**痛点**：
1. **切换即丢失**（最痛）：当前 `<router-view>` 在路由切换时会**销毁上一个页面实例**。运营常需"一边在发布文章里写稿、一边去工作台/草稿箱核对"，一切回发布文章，表单被重新加载、已填内容清空。对内容发布工具这是高频硬伤。
2. **创作中心与主界面割裂**：账号工作台是独立 OS 窗口，无法与系统面板（账号管理 / 一键发布）在同一界面并行，多窗口来回切。
3. **隔离心智不可见**：每个账号对应哪个隔离环境（代理/UA）无统一可见标识。

**业务目标**：在**保留左侧竖栏**的前提下，于右侧内容区最顶部新增一条「任务选项卡栏」——它是一组"当前打开的工作台 / 账号创作中心"集合；切换 tab **不重新加载、保留已填内容**；账号创作中心以全局任务 tab 形式内嵌进主窗口（替代独立弹窗），其内部沿用已有的账号内子页签。

---

## 2. 目标与成功指标

| 维度 | 目标 |
|---|---|
| 状态保持 | 在任务 tab 间切换，页面已填表单 / 滚动位置 / 未提交内容 **100% 保留**（重点验证：发布文章） |
| 窗口收敛 | 账号创作中心从独立 OS 窗口 → 主窗口内的任务 tab，运营所需窗口数 1（主窗口） |
| 切换效率 | 系统面板与账号创作中心可在同一界面并行、一键切换 |
| 安全可视化 | 账号创作中心 tab 标注其隔离环境（平台图标 + 昵称 + 🔒 环境名），串号客诉 → 0 |
| 功能等价 | 现有创作中心能力（网页多标签 / 前进后退 / 刷新 / 新开）迁入主窗口后 100% 等价 |

## 3. 产品定义

### 3.1 系统级任务选项卡（System Task Tabs）
- **位置**：右侧内容区的**最顶部**（左侧竖栏保持不动；与现有 topbar 的搜索框在**同一行**——tab 条居左、搜索框居右，详见 §12 布局）。
- **触发**：点击左侧竖栏任意项 → 在任务选项卡栏**新增 / 激活**对应任务 tab。重复点击同一项 → 仅激活已存在的 tab（不重复开）。
- **内容**：系统面板页面 —— 仪表盘 / 账号管理 / 工作台(草稿箱·浏览器) / 一键发布(视频·图文·文章·历史) / 系统配置(环境·代理·接口·日志)。每个系统路由对应**一个**任务 tab。
- **状态保持（核心）**：每个已打开的任务 tab 保持其实例**常驻存活**（隐藏而非卸载），切走再切回不重新渲染、不丢已填内容。
- **默认**：启动后「仪表盘」作为初始任务 tab 打开。✅【已确认】

### 3.2 账号创作中心全局任务选项卡（Account Creator-Center Global Tab）
- **触发**：在「账号管理」中点击某账号「进入创作中心」→ 在任务选项卡栏**新增 / 激活**「xxx 创作中心」全局任务 tab（xxx = 账号昵称/标识）。
- **内容**：该账号的平台网页工作台，**复用现有 `CreatorTabWindow` 的 WebContentsView + 隔离能力**，但**宿主从独立 `BrowserWindow` 改为主窗口内容区内的一块矩形区域**（见 §8）。
- **隔离**：沿用 `session.fromPartition('persist:account_${id}')` + `BrowserEnvService.applyEnvironment`，与现状等价。
- **标识**：tab 显示 平台图标 + 账号昵称 + 🔒 环境名。
- **默认首页**：沿用现有 `openCreator` 的 `homeUrl`（各平台创作首页，如小红书创作服务平台首页）。✅【已确认】

### 3.3 自媒体账号内子页签（Inner Page Tabs）
- **位置**：账号创作中心任务 tab 内部（顶部账号信息条下方）。
- **内容**：该平台不同功能模块并行（如小红书：创作服务平台首页 / 笔记管理 / 数据分析 / 新开网页）。
- **交互**：新增 / 关闭 / 切换 / 前进后退刷新；`target=_blank` 在**新 inner tab** 打开。
- **说明**：账号工作台为**纯内嵌网页**，不在 Vue 层渲染 FlowX 自有"粉丝看板"；平台原生数据看板由平台网页自身提供，作为 inner tab 之一即可。
- **复用**：直接沿用现有 `CreatorTabWindow` 的 createTab / activateTab / closeTab / 前进后退 / 新开逻辑，仅把宿主从独立窗口改为主窗口指定矩形。

## 4. 用户故事

- 作为运营，我希望同时打开「草稿箱」和「发布文章」两个任务 tab，在它们之间自由切换，且切回「发布文章」时我写了一半的稿子还在，以便边核对草稿边发文不丢内容。
- 作为运营，我希望「账号管理」和一个账号的「创作中心」能并排成两个任务 tab，以便一边管账号、一边盯某个账号的创作中心，不用开两个窗口。
- 作为运营，我希望在单个账号创作中心 tab 里并排切它的「笔记管理」和「数据分析」inner tab，以便对照数据不用后退重载。
- 作为安全/合规负责人，我希望每个账号创作中心 tab 明确标注其隔离环境（代理 / UA），以便确认没有串号。

## 5. 功能需求（FR）

### Must（v1 必做）
- **FR1** 左侧竖栏导航**完全保留不变**（仪表盘 / 账号管理 / 工作台 / 一键发布 / 系统配置 及其全部二级项）。
- **FR2** 右侧内容区最顶部新增「任务选项卡栏」（与现有 topbar 搜索框横向并存）。
- **FR3** 点击左侧竖栏项 → 在任务选项卡栏新增/激活对应系统任务 tab；同一路由不重复开 tab。
- **FR4** **状态保持**：任务 tab 切换时不卸载页面实例，已填表单 / 滚动 / 未提交内容 100% 保留。重点验证「发布文章」表单。✅【已确认：全部系统 tab 保活，实现统一、代价低】
- **FR5** 「账号管理」点击账号「进入创作中心」→ 新增/激活「xxx 创作中心」全局任务 tab。
- **FR6** 账号创作中心 tab 内容复用现有 `CreatorTabWindow` 的 WebContentsView + 隔离，宿主改为主窗口内容区矩形（替代独立弹窗）。
- **FR7** 账号创作中心 tab 内部复用现有 inner 子页签（新增 / 关闭 / 切换 / 前进后退刷新 / `target=_blank` 新 inner tab）。
- **FR8** 账号创作中心 tab 显示隔离环境标识（平台图标 + 昵称 + 🔒 环境名），与账号管理列表环境标注一致。
- **FR9** 任务 tab 支持关闭（×）；关闭后回退到相邻 tab；最后一个系统 tab 关闭后回到默认（仪表盘）。
- **FR10** 全局任务 tab 数量上限 **≤ 20**；超过时禁用「+」并提示。✅【已确认】

### Should（v1 建议）
- 任务选项卡打开状态**持久化/恢复**（重开 App 恢复上次 tab 集，至少恢复账号创作中心 tab）。
- 「发布文章」类 tab 含未提交内容时，关闭前**确认**（"内容未保存，确定关闭？"）。
- 任务选项卡栏可横向滚动（tab 多时，两侧渐隐遮罩）。

### Could（后续）
- tab 拖拽重排、中键关闭、双击关闭。
- 账号创作中心 tab "在独立窗口打开"高级入口（过渡期可选；本期目标是收敛进主窗口，默认不做）。

## 6. Non-goals（本期明确不做）

- ❌ **不移除、不改动左侧竖栏导航**。
- ❌ **不在**账号创作中心渲染 FlowX 自有"粉丝看板"（纯内嵌网页，平台看板由平台网页自身提供）。
- ❌ 不做 tab 拖出成独立窗口 / 多窗口多实例（本期目标是把创作中心收敛进主窗口）。
- ❌ 不做跨账号会话合并或共享登录。
- ❌ 不做实时协同 / 多用户。

## 8. 技术实现要点

**两大宿主改造**（核心改动）：
1. **系统面板 → 状态保持的常驻 tab**：现状 = `<router-view>` 切换即销毁。目标 = 新增任务 tab 状态（Pinia store），每个已开系统 tab 的 Vue 组件实例**保活**（如用 `v-show` 显隐或 `<KeepAlive include>` 路由缓存），切换 tab = 隐藏/显示而非卸载/重挂载，从而保留表单与滚动。
2. **账号创作中心 → 主窗口内嵌 webview**：复用 `CreatorTabWindow` 的 `WebContentsView` + 隔离 + inner tab 逻辑，但**宿主从独立 `BrowserWindow` 改为主窗口内容区内矩形**（`mainWindow.contentView.addChildView(view)` + `setBounds`），移除独立窗口创建与 `openCreator` 弹窗逻辑。

具体：
1. **任务 tab 状态 store**：`tabs: {id, kind:'system'|'account', route?, accountId?, title, envId?, mounted?}[]` + `activeId`，负责增删/激活/持久化。
2. **系统 tab 渲染**：`<router-view>` 改为"按 active tab 渲染对应组件，所有已开 tab 组件实例保活"。发布文章等表单页状态天然保留。
3. **账号 tab 渲染**：主进程据 active 账号 tab 计算内容矩形，在区域内 hosting `WebContentsView`；inner tabs 逻辑沿用 `CreatorTabWindow`（宿主改为主窗口指定矩形）。
4. **隔离复用**：`session.fromPartition('persist:account_${id}')` + `BrowserEnvService.applyEnvironment` 不变。
5. **移除**：`CreatorTabWindow` 独立窗口创建 / `openCreator` 弹窗；`electronApi.openCreator` 改为 `openAccountTab(accountId)`（通知主进程增/激活账号任务 tab）。
6. **布局同步**：主进程监听主窗口 resize / active tab 切换，重算并 `setBounds`；非激活账号 tab 的 view 需 `setVisible(false)` 或 `removeChildView`，避免遮挡 Vue UI。
7. **资源**：非激活账号 webview 可隐藏而非销毁；系统 tab 为轻量 Vue 组件，常驻内存代价低。

## 9. 风险与依赖

| 风险 | 说明 | 缓解 |
|---|---|---|
| R1 状态保持实现正确性 | 若用 KeepAlive 缓存路由，需确保表单数据/未提交态确实保留；router-view 默认行为会丢 | 采用组件实例保活（v-show）或显式 KeepAlive include；以「发布文章」表单做回归验证 |

| 依赖 | 合规 P0 已落地不受影响；BrowserEnvService 隔离能力直接复用；现有 CreatorTabWindow 的 inner tab 逻辑直接复用 | — |

### 11.3 组件拆解与视觉规格

**A. 任务选项卡条（容器）**
- 高度 48px，内边距 `0 12px 0 8px`，flex 行，align-center。
- 左区 `flex:1 overflow-x:auto`（隐藏滚动条或细滚动条），右区搜索框固定。

**B. 系统任务 tab 项（System Tab）**
- 尺寸：高 32px，圆角 `--r-sm`(10px)（顶部略平，可做成上圆下直），padding `0 12px`，min-width 88px，max-width 180px（超出 ellipsis）。
- 文字：`--slate` 13px / 600；激活态 `--ink` 700。
- 状态：
  - 默认(非激活)：背景透明，hover 背景 `rgba(99,102,241,0.06)`。
  - 激活：背景 `--surface`，下方 2px `--brand-grad` 指示条（或整体 `--brand-grad-soft` 浅底 + `--brand-indigo` 文字）。
  - 含未保存内容：tab 名前加一个 6px `--warning` 实心点（如 `● 发布文章`）。
- 关闭按钮 ×：hover tab 时显示于右侧，12px，点击关闭（不激活时也可关）。

**C. 账号创作中心 tab 项（Account Tab）**
- 同 B 尺寸，但内容结构：`[平台图标 16px] [昵称 ellipsis] [🔒 环境名徽章] [×]`。
- 平台图标：用平台色圆点/emoji（抖音 `#000`/🎵、小红书 `#ff2442`/📕、微信 💬 等）。
- 🔒 环境名徽章：高度 18px pill，背景 `rgba(16,185,129,0.12)`，文字 `--success` 11px，`🔒 隔离-环境名`（如 `🔒 代理A·UA伪装`）。
- 激活态同 B（浅靛蓝底 + 平台色左条）。

**D. 「+」新建 tab 按钮**
- 24×24 圆形/方按钮，居中「+」18px `--muted`，hover `--brand-indigo` + 浅底。
- 达 20 个上限时禁用（灰显），点击 Toast「最多同时打开 20 个任务页」。

**E. 账号信息条（仅账号 tab 内）**
- 高度 40px，位于账号 tab 内容顶部，背景 `--surface-2`，底部 1px `--line`。
- 内容：`[头像 28px] [昵称 14px/700 --ink] [平台·粉丝数 12px --muted] ……右侧 [🔒 隔离环境全称] [刷新] [在浏览器打开]`。
- 刷新/打开按钮用 16px 图标按钮（hover 浅底）。

**F. 账号内子页签条（Inner Tabs）**
- 高度 36px，位于账号信息条下方，背景 `--surface`。
- 子 tab 项：高 28px，圆角 8px，padding `0 10px`，文字 `--slate` 12.5px；激活态 `--brand-indigo` 700 + 底部 2px `--brand-indigo` 指示。
- 右侧「+ 新开网页」文字按钮（12px `--muted`）。
- 内容区 = `WebContentsView`（占满剩余高度，物理隔离）。

### 11.5 状态与交互规格

| 元素 | 交互 | 视觉反馈 |
|---|---|---|
| 任务 tab | 点击 | 激活（浅靛蓝底 + 指示条），对应内容渲染/显示，旧 tab 实例隐藏保活 |
| 左侧菜单项 | 点击已开 tab 的对应项 | 仅激活已有 tab，不新增 |
| tab × | 点击 | 关闭该 tab；若为激活 tab，聚焦相邻 tab；最后系统 tab 关→回仪表盘 |
| 含未保存 tab × | 点击 | 弹确认「内容未保存，确定关闭？」[取消][关闭] |
| + 按钮 | 点击 | 新增空白/默认 tab（或弹出"打开面板"菜单，列系统面板项） |
| 账号 tab | hover | 显示 🔒 环境徽章 tooltip 明细（代理/IP/UA） |
| inner tab | 点击/关闭/+/前进后退 | 复用现有 CreatorTabWindow 行为 |
| tab 条溢出 | 横向滚动 | 两侧 24px 渐隐遮罩（--surface → transparent） |
| 搜索框 | 输入 | 复用现有搜索交互（全局搜索/过滤） |

### 11.6 边界场景

1. **tab 溢出**：> 视口宽度时横向滚动 + 两侧渐隐遮罩；达 20 上限禁用「+」+ Toast。
2. **未保存关闭**：含未提交内容的系统 tab（发布文章/草稿箱编辑中）关闭前确认。
3. **空状态**：全部系统 tab 关闭 → 自动回仪表盘默认 tab（不应出现空白条）。
4. **账号 tab 隔离失效兜底**：若 `BrowserEnvService` 未配置环境，🔒 徽章显示「未配置隔离」(--warning) 而非隐藏，避免误以为已隔离。
5. **启动恢复**：重开 App 恢复上次 tab 集（至少恢复账号创作中心 tab），恢复时仪表盘默认激活。
